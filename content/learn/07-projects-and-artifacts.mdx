# 프로젝트와 아티팩트 활용하기

Claude를 어느 정도 써보다 보면 불편한 점이 하나 생겨요. 새 대화를 열 때마다 배경 설명을 처음부터 다시 해야 한다는 거예요. "저는 식품 스타트업을 운영 중이고, 타겟은 30대 건강 관심 직장인이고, 브랜드 톤은 친근하고 전문적이에요…" 이 설명을 매번 반복하는 건 정말 번거롭죠. 그래서 등장한 게 **Projects** 기능이에요.

## Projects 기능이란?

**Projects**는 여러 대화에 걸쳐 맥락을 유지할 수 있는 일종의 폴더예요. 프로젝트 안에 저장해둔 정보는 그 프로젝트 내의 모든 대화에 자동으로 적용돼요.

### 언제 유용한가요?

- **같은 주제로 반복 작업할 때**: 블로그 운영, 특정 고객 대응, 마케팅 캠페인 등
- **팀 스타일 가이드가 있을 때**: 브랜드 보이스, 글쓰기 규칙을 한 번만 등록해두면 매번 설명 불필요
- **긴 프로젝트를 여러 단계로 진행할 때**: 사업계획서, 책 집필, 장기 리서치 작업 등

### 프로젝트 만드는 방법

<StepByStep>
  <Step number={1}>Claude.ai 왼쪽 사이드바에서 "Projects" 섹션을 찾으세요.</Step>
  <Step number={2}>"New project" 버튼을 클릭하세요.</Step>
  <Step number={3}>프로젝트 이름을 입력하세요. 예: "마케팅 블로그", "고객 제안서", "사이드 프로젝트"</Step>
  <Step number={4}>프로젝트 Instructions(지침)에 Claude가 이 프로젝트에서 항상 알아야 할 내용을 입력하세요.</Step>
  <Step number={5}>이제 이 프로젝트 안에서 새 대화를 시작하면 지침이 자동으로 적용돼요.</Step>
</StepByStep>

### 프로젝트 Instructions에 넣으면 좋은 것들

- 브랜드/서비스 소개
- 타겟 고객 정보
- 글쓰기 스타일 규칙 (격식체/반말, 특정 단어 사용 규칙 등)
- 자주 쓰는 형식이나 템플릿
- 피해야 할 표현이나 금지 사항

### 프로젝트에 파일 업로드하기

프로젝트에 PDF, Word 문서, 텍스트 파일 등을 업로드해두면 Claude가 그 파일의 내용을 참고해서 답변해줘요. 예를 들어:

- 회사 소개서를 업로드해두면 관련 문서 작성 시 일관된 내용 사용
- 브랜드 가이드라인 PDF를 올려두면 글쓰기에 자동 반영
- 제품 스펙 문서를 올려두면 Q&A 답변 생성에 활용

<Callout type="info">Projects 기능은 Claude Pro 이상에서 사용 가능해요. 무료 플랜에서는 사용이 제한될 수 있어요.</Callout>

## Artifacts 기능이란?

**Artifacts**는 Claude가 생성한 결과물을 대화창 오른쪽 별도 패널에 표시하는 기능이에요. 대화가 길어져도 결과물이 흘러가지 않고 패널에 고정되어 있어서 편리해요.

### 어떤 것들이 Artifact로 표시되나요?

- **코드**: Python, JavaScript 등 프로그래밍 코드
- **HTML**: 웹 페이지나 간단한 인터페이스
- **마크다운**: 긴 문서, 보고서, README 파일
- **SVG**: 벡터 이미지나 간단한 다이어그램
- **React 컴포넌트**: 인터랙티브한 UI 요소

### Artifact 활용 방법

Artifact 패널에는 몇 가지 유용한 버튼이 있어요:

- **복사**: Artifact 내용을 클립보드에 복사
- **다운로드**: 파일로 저장
- **미리보기**: HTML이나 React 코드의 경우 실제 화면을 렌더링해서 보여줌

코드 Artifact의 경우 "Preview" 탭을 클릭하면 실제 결과물을 바로 확인할 수 있어요. 개발자가 아니어도 간단한 계산기나 HTML 페이지를 Claude가 만들어주면 미리보기로 바로 확인할 수 있죠.

## 실전 활용 팁

**블로그 운영 예시:**
블로그 프로젝트를 만들고 Instructions에 다음 내용을 넣어두세요:
- 블로그 이름과 주제
- 타겟 독자 (예: "직장인 30-40대, 투자 입문자")
- 글쓰기 스타일 (예: "친근하고 전문적, 은어 사용 금지")
- 자주 쓰는 구조 (예: "도입부 → 본론 3가지 → 마무리 정리")

이렇게 설정해두면 새 포스팅을 쓸 때마다 "오늘 주제는 ~~야. 평소 스타일로 써줘"라고만 해도 일관된 블로그 글이 나와요.

<Callout type="tip">프로젝트를 쓴다면 처음에 시간을 들여 Instructions를 잘 작성해두는 게 핵심이에요. 나중에 수백 번의 반복 설명을 아낄 수 있어요.</Callout>

Projects와 Artifacts를 잘 활용하면 Claude가 단순한 채팅 도구가 아니라 진짜 작업 파트너처럼 느껴지기 시작해요. 반복적인 작업이 있다면 꼭 프로젝트를 만들어보세요!
