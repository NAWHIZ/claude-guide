# 슬래시 커맨드 활용하기

Claude Code에는 `/`로 시작하는 특별한 명령어들이 있어요. 이 명령어들을 알면 Claude Code를 훨씬 더 효율적으로 쓸 수 있어요.

## 슬래시 커맨드란?

슬래시 커맨드는 Claude Code에게 특정 동작을 지시하는 특수 명령어예요. 일반 대화와 달리 `/`로 시작하고, Claude에게 "이런저런 작업을 해줘"라고 부탁하는 게 아니라 Claude Code 프로그램 자체에 직접 명령을 내리는 방식이에요.

예를 들어 `/clear`는 "지금까지 나눈 대화를 지우고 새로 시작해"라는 뜻이고, `/help`는 "사용 가능한 명령어들을 보여줘"라는 뜻이에요.

## 주요 슬래시 커맨드

### `/help` — 도움말 보기

```
> /help
```

사용 가능한 모든 슬래시 커맨드 목록과 간단한 설명을 보여줘요. 뭔가 궁금하거나 어떤 명령어가 있는지 확인하고 싶을 때 가장 먼저 써보세요.

### `/clear` — 대화 기록 초기화

```
> /clear
```

지금까지 나눈 대화 내용을 모두 지우고 새로 시작해요. 새로운 주제나 다른 프로젝트 작업을 시작할 때 유용해요. 대화를 지운다고 해서 파일이 변경되거나 하는 건 아니에요 — 단지 Claude의 "기억"을 초기화하는 거예요.

### `/compact` — 대화 요약해서 컨텍스트 절약

```
> /compact
```

대화가 길어지면 Claude Code가 처리해야 하는 정보량이 많아져서 느려지거나 비용이 늘어날 수 있어요. `/compact`는 긴 대화 내용을 핵심만 남겨서 요약해줘요. 작업의 흐름은 유지하면서 컨텍스트를 효율적으로 줄여요.

### `/exit` 또는 `/quit` — 종료

```
> /exit
```

Claude Code를 종료해요. `Ctrl+C`를 눌러도 같은 효과예요.

### `/undo` — 마지막 변경 취소

```
> /undo
```

Claude Code가 마지막으로 한 파일 변경 작업을 취소해요. 수정 결과가 마음에 들지 않을 때 유용해요. 단, 모든 변경 사항이 취소 가능한 것은 아니므로 중요한 파일은 미리 백업해두는 게 좋아요.

### `/review` — 변경사항 검토

```
> /review
```

Claude Code가 현재 세션에서 변경한 내용들을 정리해서 보여줘요. 작업을 끝내기 전에 무엇이 바뀌었는지 확인할 때 써요.

### `/model` — 사용 모델 변경

```
> /model
```

Claude Code가 사용하는 AI 모델을 바꿀 수 있어요. 빠르고 저렴한 모델(Claude Haiku)과 더 강력한 모델(Claude Sonnet, Opus) 중에서 선택할 수 있어요.

## 언제 어떤 커맨드를 쓰면 좋을까?

각 상황에 맞는 커맨드를 선택하면 작업이 훨씬 효율적이에요.

| 상황 | 사용할 커맨드 |
|------|------------|
| 새로운 프로젝트나 주제로 넘어갈 때 | `/clear` |
| 대화가 너무 길어져서 느려졌을 때 | `/compact` |
| 수정한 결과가 마음에 안 들 때 | `/undo` |
| 무슨 커맨드가 있는지 잊어버렸을 때 | `/help` |
| 작업을 마무리하기 전에 확인할 때 | `/review` |
| 간단한 작업에 비용을 아끼고 싶을 때 | `/model` 로 Haiku 선택 |

## 실용적인 팁

<Callout type="tip">
모르면 `/help`를 먼저 쳐보세요. Claude Code가 현재 사용 가능한 모든 명령어와 기능을 알려줘요. 새 버전에서 추가된 기능도 여기서 확인할 수 있어요.
</Callout>

### 작업 흐름에 통합하기

새 프로젝트를 시작할 때는 이런 순서로 해보세요.

1. 프로젝트 폴더로 이동
2. `claude` 실행
3. 작업 진행
4. 다른 프로젝트로 넘어갈 때 `/clear`
5. 작업이 완료되면 `/review`로 변경 사항 확인
6. `/exit`로 종료

### 비용 절약을 위한 `/compact` 활용

같은 프로젝트에서 오랫동안 작업할 때는 중간중간 `/compact`를 써서 대화를 압축해두면 API 비용을 줄일 수 있어요. 특히 파일이 많은 프로젝트에서는 대화가 금방 길어지거든요.
