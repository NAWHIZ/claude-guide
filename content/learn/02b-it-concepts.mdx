# 개발자가 쓰는 말, 비유로 이해하기

이 가이드를 따라가다 보면 낯선 단어들이 자주 등장해요. **GitHub**, **API**, **Vercel**, **환경변수**... 처음 보면 외계어처럼 느껴지지만, 사실 모두 일상에서 쉽게 찾을 수 있는 개념이에요.

지금 다 외울 필요 없어요. "아, 이런 거구나" 하고 감만 잡으면 됩니다. 나중에 막히면 이 페이지로 다시 돌아오세요.

---

## API — 식당의 웨이터

**API**는 두 서비스가 서로 대화하는 방식이에요.

> **비유**: 식당에서 손님(내 앱)이 주방(Claude, 날씨 서비스 등)에 직접 들어갈 수 없잖아요. 웨이터(API)가 주문을 전달하고, 음식을 가져다 줘요. 정해진 메뉴(규칙) 안에서만 요청할 수 있고, 주방이 어떻게 만드는지는 몰라도 됩니다.

Claude Code가 Claude AI한테 질문을 보내고 답을 받는 것도 API예요. 내가 어떤 서비스를 만들어서 날씨 정보를 보여주고 싶다면? 날씨 API를 연결하면 돼요.

---

## API 키 — 회원 카드

API를 쓰려면 **API 키**가 필요해요. 내가 누구인지 증명하는 열쇠예요.

> **비유**: 헬스장 회원 카드예요. 카드가 있어야 입장 가능하고, 카드로 내 이용 내역이 기록돼요. 누군가 내 카드를 훔치면 내 이름으로 헬스장을 쓸 수 있겠죠. API 키도 마찬가지라 절대 공개하면 안 돼요.

---

## 환경변수 — 열쇠를 서랍에 보관하기

API 키 같은 민감한 정보를 코드 안에 직접 적지 않고, **별도의 파일에 따로 보관**하는 방식이에요.

> **비유**: 집 열쇠를 현관문에 붙여두는 사람은 없어요. 서랍에 따로 보관하죠. 환경변수는 그 서랍이에요. 코드(현관문)엔 "열쇠는 서랍에 있어"라고만 적고, 실제 열쇠(.env 파일)는 따로 보관합니다. GitHub에 코드를 올릴 때도 이 서랍은 올리지 않아요.

---

## GitHub — 구글 드라이브 + 타임머신

코드를 저장하고, 변경 내역을 관리하는 **클라우드 저장소**예요.

> **비유**: 구글 드라이브처럼 파일을 저장하는데, 타임머신 기능이 있어요. 언제 누가 무엇을 바꿨는지 기록이 다 남아요. "어제 버전으로 돌려줘"가 가능해요. 여러 사람이 같은 파일을 동시에 수정해도 충돌을 정리해줘요.

Claude Code에게 "GitHub에 올려줘"라고 하면 자동으로 저장해줘요.

---

## Vercel — 웹사이트 간판 설치 업체

만든 웹사이트를 **인터넷에 공개**해주는 서비스예요.

> **비유**: 제가 만든 간판(웹사이트)을 실제 도로변(인터넷)에 설치해주는 업체예요. 간판이 아무리 잘 만들어져도 창고에만 있으면 아무도 못 봐요. Vercel이 그걸 전 세계 누구나 볼 수 있는 자리에 설치해줍니다. GitHub에 새 코드를 올리면 자동으로 새 간판으로 교체까지 해줘요.

배포가 끝나면 `내프로젝트.vercel.app` 주소가 생겨요.

---

## Supabase — 인터넷 위의 엑셀

웹서비스의 **데이터를 저장하고 꺼내오는** 온라인 데이터베이스예요.

> **비유**: 엑셀 스프레드시트가 인터넷 위에 있는 것과 같아요. 차이점은 여러 사람이 동시에 쓸 수 있고, 내 웹사이트가 실시간으로 데이터를 주고받을 수 있다는 거예요. 누군가 회원가입하면 그 정보가 여기 저장되고, 게시글을 쓰면 여기 쌓여요.

---

## RLS — 사무실 출입 카드

Supabase의 **데이터 접근 권한 관리** 시스템이에요.

> **비유**: 회사 출입 카드예요. 일반 직원은 사무실만, 관리자는 서버실까지 들어갈 수 있죠. RLS는 데이터베이스에서 "이 데이터는 로그인한 사람만", "저 데이터는 누구나 볼 수 있어" 같은 규칙을 설정하는 거예요. 이걸 안 설정하면 아무도 데이터를 못 읽어요 — 열쇠 정책 없는 금고처럼요.

---

## MCP — 스마트폰 앱

Claude Code가 **외부 서비스를 직접 다룰 수 있게** 연결해주는 플러그인이에요.

> **비유**: 기본 스마트폰에 카카오톡 앱을 설치하면 메시지를 보낼 수 있듯이, Claude Code에 Figma MCP를 설치하면 Claude가 Figma 파일을 직접 열어볼 수 있어요. Notion MCP를 달면 Notion 페이지를 읽고 쓸 수 있고요. 앱처럼 원하는 기능을 골라서 설치하면 됩니다.

---

## OMC (oh-my-claudecode) — 1명 vs. 팀

Claude Code 위에 설치하는 **멀티에이전트 시스템**이에요. 여러 전문가 AI가 역할을 나눠 동시에 일해요.

> **비유**: 만능 직원 1명(기본 Claude Code)이 혼자 기획·개발·검토를 다 하는 것 vs. 기획팀·개발팀·QA팀이 각자 전문 역할을 맡아 동시에 일하는 회사(OMC). 복잡한 프로젝트일수록 팀이 훨씬 빠르고 정확해요.

---

## Cron Job — 예약 알람

**정해진 시간에 자동으로 실행**되는 예약 작업이에요.

> **비유**: 스마트폰 알람이에요. 한 번 맞춰두면 매일 같은 시간에 울리는 것처럼, Cron은 서버가 매일 같은 시간에 특정 작업을 자동으로 실행해요. "매일 오전 10시에 뉴스 수집해", "매주 월요일에 리포트 만들어" 같은 걸 설정해두면 내가 아무것도 안 해도 돌아가요.

---

## 로컬 vs. 배포

**로컬(local)**: 내 컴퓨터에서만 보이는 상태. 개발 중인 상태.
**배포(deploy)**: Vercel 같은 곳에 올려서 인터넷에 공개된 상태.

> **비유**: 로컬은 요리사가 주방에서 혼자 맛보는 거예요. 배포는 그 요리를 손님 테이블에 내보내는 거고요. 아직 맛이 덜 됐을 때는 주방(로컬)에서만 확인하고, 완성됐을 때 손님(인터넷)한테 내보내요.

---

## 오픈소스 vs. 클로즈드소스

**오픈소스**: 코드가 공개되어 있어서 누구나 볼 수 있고, 기여할 수 있어요.
**클로즈드소스**: 코드가 비공개예요. 결과물만 쓸 수 있어요.

> **비유**: 오픈소스는 레시피가 공개된 식당이에요. 누구나 레시피를 보고, 개선하고, 자기 식당에서 써도 돼요. 클로즈드소스는 비밀 레시피로 운영하는 식당이고요. Claude Code는 클로즈드소스, GitHub은 오픈소스 코드를 공유하는 플랫폼으로도 쓰여요.
