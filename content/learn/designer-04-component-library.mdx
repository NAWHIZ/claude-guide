# 컴포넌트 라이브러리 자동 문서화

팀에서 "이 버튼 디자인이 맞아요?"라는 질문이 반복된다면 컴포넌트 라이브러리 사이트가 필요한 거예요. Storybook 설정에 며칠을 쏟지 않아도 됩니다 — Claude Code가 훨씬 빠르게 만들어줘요.

---

## 컴포넌트 라이브러리 사이트가 필요한 이유

| 문제 | 해결 |
|------|------|
| "어떤 버튼을 써야 하지?" | 모든 버튼 변형을 한눈에 보여주는 페이지 |
| "이 색상 코드가 뭐야?" | 컬러 팔레트 페이지 |
| "폰트 크기가 몇 px이었지?" | 타이포그래피 스케일 페이지 |
| "개발팀과 같은 용어를 써야 하는데" | 컴포넌트명 통일 |

---

## 1단계: 컴포넌트 인벤토리 만들기

```
> 우리 프로젝트의 components/ 폴더를 스캔해서
  컴포넌트 인벤토리를 만들어줘.
  
  각 컴포넌트별:
  - 컴포넌트명
  - Props 목록 (TypeScript 타입에서 추출)
  - 사용 위치 (어느 페이지에서 쓰이는지)
  - 의존 컴포넌트
  
  component-inventory.md로 저장.
```

---

## 2단계: 컴포넌트 쇼케이스 사이트 생성

```
> Next.js로 컴포넌트 라이브러리 쇼케이스 사이트를 만들어줘.
  
  페이지 구성:
  - /components: 모든 컴포넌트 목록
  - /components/[name]: 개별 컴포넌트 상세
  - /colors: 컬러 팔레트
  - /typography: 타이포그래피 스케일
  - /icons: 아이콘 목록
  
  각 컴포넌트 페이지:
  - 실제 렌더링 (모든 variant 표시)
  - Props 테이블
  - 코드 스니펫 (복사 버튼 포함)
  - Figma 링크
```

---

## 3단계: Figma와 연동

```
> 컴포넌트 라이브러리 페이지에
  각 컴포넌트의 Figma 링크와 미리보기를 추가해줘.
  
  Figma 파일: https://www.figma.com/design/XXXXX
  
  각 컴포넌트 카드에:
  - Figma 썸네일 이미지
  - "Figma에서 열기" 링크
  - 마지막 업데이트 날짜
```

---

## 4단계: 자동 문서화 파이프라인

컴포넌트가 업데이트될 때 문서도 자동으로 업데이트:

```
> components/ 폴더의 TypeScript 파일을 읽고
  각 컴포넌트의 JSDoc 주석을 추출해서
  자동으로 문서를 생성하는 스크립트를 만들어줘.
  
  없는 주석은 코드를 분석해서 자동으로 생성.
  
  스크립트: scripts/generate-docs.ts
```

---

## 5단계: 디자인 토큰 페이지

```
> Figma에서 디자인 토큰을 읽어서
  인터랙티브 디자인 토큰 페이지를 만들어줘.
  
  섹션:
  1. 컬러 팔레트 (스와치 + HEX + 사용 용도)
  2. 타이포그래피 (폰트 크기 + weight + line-height + 실제 미리보기)
  3. 간격 (spacing scale + 시각적 표현)
  4. 그림자 (shadow scale + 미리보기)
  5. 보더 라디우스 (미리보기)
  
  각 토큰에 CSS 변수 이름과 Tailwind 클래스 표시.
```

---

## 6단계: 팀 공유 및 배포

```
> 컴포넌트 라이브러리 사이트를 Vercel에 배포하고
  비밀번호로 보호해줘.
  
  비밀번호는 환경변수 LIBRARY_PASSWORD로 설정.
  팀 내부용이라 외부 공개 필요 없음.
```

<Callout type="tip">
컴포넌트 라이브러리 사이트 URL을 팀 Slack 채널 북마크에 추가해두세요. "어떤 버튼 써요?" 질문이 사라질 거예요.
</Callout>

---

## 접근성 자동 테스트 추가

```
> 컴포넌트 라이브러리에 접근성 자동 테스트를 추가해줘.
  
  각 컴포넌트 페이지에:
  - 색상 대비 비율 자동 계산
  - 키보드 포커스 동작 확인
  - aria 속성 검사
  
  문제가 있는 컴포넌트는 빨간 배지 표시.
```
