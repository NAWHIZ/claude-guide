# 고급 활용 테크닉

Claude의 기본 사용법을 익혔다면 이제 한 단계 더 깊이 들어가볼 차례예요. 시스템 프롬프트, 컨텍스트 윈도우 관리, Extended Thinking, API 연동까지 — 이 기능들을 이해하면 Claude를 훨씬 더 강력하게 활용할 수 있어요.

## 시스템 프롬프트 (System Prompt)

### 시스템 프롬프트가 뭔가요?

시스템 프롬프트는 대화 전체에 걸쳐 적용되는 기본 설정 지시문이에요. 일반 메시지와 달리, 시스템 프롬프트는 Claude의 기본 행동 방식을 바꿔요. 마치 "이 대화에서는 항상 이렇게 행동해"라고 미리 약속해두는 것과 같아요.

예를 들어 이런 시스템 프롬프트를 설정할 수 있어요:

> "당신은 전문 카피라이터입니다. 항상 간결하고 임팩트 있는 문장을 사용하세요. 불필요한 수식어는 피하고, 행동을 유도하는 표현을 포함하세요. 답변은 항상 한국어로 해주세요."

이 설정이 되어 있으면 매번 "카피라이터처럼 써줘"라고 하지 않아도 자동으로 그 스타일로 답변해요.

### Claude.ai에서 시스템 프롬프트 사용하기

Claude.ai에서는 **Projects의 Custom Instructions**가 시스템 프롬프트 역할을 해요. 프로젝트를 만들고 Instructions 섹션에 원하는 지시 내용을 입력하면 돼요.

시스템 프롬프트에 담으면 좋은 것들:
- 역할과 전문성 ("당신은 ~~전문가입니다")
- 말투와 스타일 ("항상 반말로", "격식체 유지")
- 형식 규칙 ("답변은 항상 bullet point로", "영어 용어는 한국어로 설명 추가")
- 금지 사항 ("불확실한 내용은 추측하지 말고 모른다고 해주세요")
- 특정 정보 주입 ("우리 회사 이름은 ~~이고, 주요 제품은 ~~입니다")

## 컨텍스트 윈도우 이해하기

### 컨텍스트 윈도우가 뭔가요?

Claude가 한 번에 처리할 수 있는 텍스트의 양을 **컨텍스트 윈도우**라고 해요. Claude는 현재 최대 수십만 토큰(대략 수십만 단어)을 처리할 수 있어요. 엄청나게 큰 것 같지만, 긴 대화를 계속하다 보면 한계에 다가갈 수 있어요.

### 컨텍스트 윈도우가 꽉 차면 어떻게 되나요?

대화가 아주 길어지면 Claude가 초반 내용을 "잊기" 시작할 수 있어요. 마치 긴 회의에서 초반에 나온 이야기를 잊어버리는 것처럼요. 갑자기 앞에서 정의한 내용을 모르는 것처럼 행동하거나, 일관성이 떨어지는 답변이 나온다면 이 현상일 수 있어요.

### 해결 방법

- **핵심 정보 요약해서 다시 제공**: "지금까지 우리가 결정한 사항을 정리하자면: [요약]"으로 중간에 맥락을 리프레시
- **새 대화 시작 + 요약 붙여넣기**: 대화가 너무 길어지면 새 대화를 시작하고 지금까지의 핵심 내용을 요약해서 맨 앞에 붙여넣기
- **Projects 활용**: 프로젝트 Instructions에 핵심 배경 정보를 저장해두면 매번 말하지 않아도 됨

## Extended Thinking (Claude 3.7 이상)

### Extended Thinking이 뭔가요?

Extended Thinking은 Claude가 답하기 전에 더 깊이, 더 오래 생각하는 모드예요. 일반적으로 Claude는 바로 답변을 생성하지만, Extended Thinking 모드에서는 마치 종이에 메모하며 풀듯이 단계적으로 문제를 분석한 뒤 최종 답을 제시해요.

### 언제 쓰면 좋나요?

- **복잡한 수학 문제**: 여러 단계가 필요한 계산
- **논리 추론**: "만약 A가 사실이면 B는 어떻게 되나?" 같은 복잡한 추론
- **전략적 의사결정**: 여러 요소를 고려한 최선의 선택
- **코드 디버깅**: 복잡한 버그의 근본 원인 파악
- **심층 분석**: 여러 관점에서 꼼꼼히 검토가 필요한 내용

### 활성화 방법

Claude.ai에서는 대화 입력창 근처의 설정 아이콘이나 모델 선택 메뉴에서 "Extended thinking" 또는 "Think" 옵션을 켤 수 있어요. 답변이 나오기 전에 Claude의 생각 과정이 펼쳐지는 것을 볼 수 있어요.

<Callout type="tip">Extended Thinking은 빠른 답변이 필요한 간단한 질문에는 오히려 느리게 느껴질 수 있어요. 복잡한 문제에만 켜는 게 효율적이에요.</Callout>

## Claude API 맛보기

### API가 뭔가요?

API(Application Programming Interface)는 프로그램에서 Claude를 직접 사용하는 방법이에요. 웹사이트나 앱에 Claude를 내장하거나, 자동화 워크플로에 Claude를 연결할 때 사용해요. 개발자들이 주로 쓰는 방법이에요.

### 개발자 없이 Claude API를 활용하는 방법

직접 코딩하지 않아도 Claude를 자동화에 활용할 수 있어요.

**Zapier 연동**: 이메일이 오면 자동으로 Claude가 요약해서 Slack에 보내기, 구글 폼 제출 시 Claude가 자동으로 맞춤 답변 생성하기 등의 자동화를 코딩 없이 만들 수 있어요.

**Make (구 Integromat) 연동**: Zapier보다 더 복잡한 자동화를 시각적으로 설계할 수 있어요. 여러 서비스를 연결하는 복잡한 워크플로를 구성할 때 유용해요.

**노코드 AI 빌더**: Voiceflow, Botpress 같은 도구를 쓰면 Claude를 기반으로 한 챗봇을 코딩 없이 만들 수 있어요.

<Callout type="tip">API를 직접 다루지 않아도 Make(구 Integromat)나 Zapier로 Claude를 자동화에 활용할 수 있어요. 예를 들어 "새 이메일이 오면 Claude가 요약해서 알림 보내기" 같은 자동화를 코딩 없이 만들 수 있어요.</Callout>

## 활용 수준별 로드맵

지금까지 배운 것들을 단계별로 정리해봤어요:

**입문 단계**: Claude.ai에서 기본 대화, 프롬프트 기초 익히기

**중급 단계**: 역할 부여, 맥락 제공, 단계별 지시, Projects + Artifacts 활용

**고급 단계**: 시스템 프롬프트 설계, 컨텍스트 윈도우 관리, Extended Thinking 활용

**전문가 단계**: API 연동, 자동화 파이프라인 구축, 커스텀 AI 도구 개발

지금 어느 단계에 있든 괜찮아요. Claude는 처음 쓰는 사람도, 매일 업무에 쓰는 전문가도 모두를 위한 도구예요. 하나씩 익혀가다 보면 어느새 고급 기능도 자연스럽게 쓰게 될 거예요.
