# 사용자 인터뷰 분석, AI가 대신해줘

10개의 사용자 인터뷰를 분석하려면 보통 하루가 걸려요. 녹취록 읽기, 패턴 찾기, 인사이트 정리, 공유 문서 작성까지. Claude Code는 이 과정을 30분으로 줄여줍니다.

<Callout type="tip">
인터뷰 텍스트 파일만 있으면 됩니다. 녹음을 텍스트로 변환하는 건 Whisper AI나 Clova Note 같은 서비스를 써보세요.
</Callout>

---

## 준비물

- 인터뷰 텍스트 파일 (`.txt` 또는 `.md`)
- 각 인터뷰별로 별도 파일로 정리 (예: `interview-01.txt`, `interview-02.txt`)

---

## 1단계: 단일 인터뷰 분석

```
> interview-01.txt를 읽고 분석해줘.
  
  추출 항목:
  1. 핵심 Pain Points (사용자가 힘들다고 한 것들)
  2. 현재 해결 방법 (지금 어떻게 하고 있는지)
  3. 원하는 것 (명시적 요청)
  4. 숨겨진 니즈 (말하지 않았지만 행간에서 읽히는 것)
  5. 인상적인 발언 (Quote worthy 문장들)
  
  결과를 interview-01-analysis.md로 저장해줘.
```

---

## 2단계: 여러 인터뷰 일괄 분석

```
> interviews/ 폴더 안의 모든 .txt 파일을 읽고
  각각 동일한 형식으로 분석해줘.
  
  각 분석 파일은 analyses/ 폴더에 저장.
  파일명: [원본파일명]-analysis.md
```

---

## 3단계: 패턴 통합 분석

개별 분석이 끝나면 전체를 종합해요:

```
> analyses/ 폴더의 모든 분석 파일을 읽고
  다음을 정리한 research-synthesis.md를 만들어줘.
  
  1. 공통 Pain Points Top 5 (언급 빈도 포함)
  2. 사용자 세그먼트 분류 (행동 패턴 기반)
  3. 우선순위 기회 영역 (Impact × Frequency)
  4. 검증된 가정 vs 뒤집어진 가정
  5. 다음 리서치에서 확인할 질문들
  6. 즉시 액션 가능한 인사이트
```

---

## 4단계: 페르소나 자동 생성

```
> research-synthesis.md를 기반으로
  3개의 사용자 페르소나를 만들어줘.
  
  각 페르소나 포함 항목:
  - 이름, 나이, 직업 (가상)
  - 핵심 특성 3가지
  - 주요 Goal
  - 주요 Frustration
  - 우리 제품과의 관계 시나리오
  
  personas.md로 저장해줘.
```

---

## 5단계: 스테이크홀더 공유 자료 만들기

```
> research-synthesis.md와 personas.md를 기반으로
  경영진/개발팀 공유용 요약 발표 자료를 만들어줘.
  
  형식: Markdown (슬라이드 구조)
  길이: 5-7 슬라이드 분량
  포함: 핵심 인사이트 3가지, 제안 액션 3가지
  
  research-presentation.md로 저장해줘.
```

<BeforeAfter>
  <Before>
    인터뷰 10개 분석: 읽기(3h) + 노트 정리(2h) + 패턴 찾기(2h) + 문서 작성(3h) = 10시간
  </Before>
  <After>
    텍스트 파일 준비(30분) + Claude Code 분석(20분) + 검토 및 수정(20분) = 70분
  </After>
</BeforeAfter>

## 실전 팁: 인터뷰 질문 설계도 AI로

다음 인터뷰를 위한 질문도 Claude Code가 도와줄 수 있어요:

```
> research-synthesis.md를 보고
  아직 검증이 필요한 가설들을 찾아줘.
  
  그 가설을 검증하기 위한
  다음 라운드 인터뷰 질문 10개를 만들어줘.
  
  질문 유형:
  - 행동 질문 (실제로 어떻게 했는지)
  - 감정 질문 (어떻게 느꼈는지)
  - 욕구 질문 (어떻게 됐으면 하는지)
```

---

## 응용: 설문 데이터 분석

인터뷰뿐만 아니라 Google Forms 설문 결과도 분석할 수 있어요:

```
> survey-results.csv를 읽고 분석해줘.
  
  1. 주요 트렌드와 패턴
  2. 예상과 다른 결과
  3. 세그먼트별 차이 (직군, 회사 규모 등)
  4. 오픈 텍스트 응답 테마 분류
  
  시각화도 함께 만들어줘 (ASCII 차트 또는 HTML).
```
