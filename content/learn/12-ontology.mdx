# 온톨로지 — 프로젝트 지식 체계 만들기

Claude Code를 쓸수록 한 가지 불편함이 생겨요 — 매번 새로 시작할 때마다 "이 프로젝트가 뭔지, 어떤 규칙이 있는지"를 다시 설명해야 한다는 거예요. 온톨로지는 이 문제를 해결합니다.

## 온톨로지가 뭔가요?

온톨로지는 원래 철학 용어인데, Claude Code에서는 **"프로젝트에 대한 지식을 구조화해서 저장하는 것"** 을 의미해요.

> **비유**: 회사에 신입직원이 들어오면 업무 매뉴얼을 줘요. 매뉴얼에는 회사 구조, 담당 업무, 용어 설명, 주의사항 등이 담겨 있죠. Claude Code의 온톨로지도 마찬가지예요 — AI 비서에게 주는 "프로젝트 완전 가이드"예요. 이걸 잘 만들어두면 Claude Code가 새로 실행될 때마다 마치 오래된 동료처럼 프로젝트를 이해하고 작업해요.

## 온톨로지의 핵심 파일들

### 1. CLAUDE.md — 프로젝트 기본 지식

가장 중요한 파일이에요. Claude Code가 시작할 때 자동으로 읽어요.

```markdown
# 프로젝트명: ABC 온라인 쇼핑몰

## 이 프로젝트가 무엇인가요?
ABC 회사의 공식 온라인 쇼핑몰입니다.
고객이 제품을 검색하고 구매할 수 있어요.

## 핵심 개념 (용어 사전)
- **상품(Product)**: 판매하는 개별 아이템
- **카테고리(Category)**: 상품 분류 (의류, 전자제품 등)
- **주문(Order)**: 고객의 구매 요청
- **재고(Stock)**: 현재 판매 가능한 수량

## 폴더 구조
- `products/` — 상품 데이터 파일들
- `orders/` — 주문 처리 관련 파일들
- `customers/` — 고객 정보 파일들
- `reports/` — 분석 보고서

## 절대 하면 안 되는 것
- 고객 개인정보(이름, 연락처)가 담긴 파일을 수정하기 전에 반드시 알려줘
- 가격 정보 변경 시 반드시 확인 요청할 것
- orders/ 폴더 파일 삭제 금지

## 파일 명명 규칙
- 날짜 포함: YYYYMMDD_파일명.확장자
- 영문 소문자, 하이픈만 사용
- 예: 20240326_product-list.csv
```

### 2. AGENTS.md — 에이전트 역할 정의

여러 에이전트를 쓸 때, 각 에이전트가 어떤 역할을 맡는지 정의하는 파일이에요.

> **비유**: 회사 조직도예요. 어떤 팀이 있고, 각 팀이 무슨 일을 하는지 명시해둬요.

```markdown
# 에이전트 역할 정의

## 데이터 분석 에이전트
- 담당: products/, orders/ 폴더 데이터 분석
- 권한: 읽기 전용 (수정 금지)
- 출력: 분석 결과를 reports/ 폴더에 저장

## 문서 작성 에이전트
- 담당: 마케팅 문서, 제품 설명 작성
- 참고 파일: templates/ 폴더의 템플릿 사용
- 톤: 친근하고 명확한 한국어

## 데이터 수정 에이전트
- 담당: 상품 정보, 가격 업데이트
- 주의: 변경 전 항상 백업 생성
- 로그: changes/ 폴더에 변경 기록 남길 것
```

### 3. 프로젝트 메모리 (.omc/project-memory.json)

oh-my-claudecode를 사용한다면 자동으로 관리되는 구조화된 메모리 파일이에요. Claude Code가 작업하면서 배운 것들을 저장해요.

```json
{
  "project": {
    "name": "ABC 쇼핑몰",
    "domain": "이커머스",
    "learned": [
      "가격 변경은 항상 팀장 승인 필요",
      "products/seasonal/ 폴더는 시즌 상품만",
      "고객 이메일은 customer_id로만 참조"
    ]
  }
}
```

## 좋은 온톨로지 만드는 방법

### 1단계: 핵심 개념 목록 만들기

프로젝트에서 자주 등장하는 핵심 용어들을 정리해요:

```
> 이 프로젝트 폴더를 살펴보고 핵심 개념과 용어들을 정리해서
  CLAUDE.md의 "핵심 개념" 섹션에 추가해줘
```

### 2단계: 폴더 구조 설명 추가

각 폴더가 무슨 역할을 하는지:

```
> 이 프로젝트의 폴더 구조를 파악하고
  각 폴더의 역할을 CLAUDE.md에 추가해줘
```

### 3단계: 규칙과 제약 명시

반드시 지켜야 할 것, 하면 안 되는 것:

```
> CLAUDE.md에 다음 규칙들을 추가해줘:
  - 고객 데이터 파일은 수정 전 항상 확인 요청
  - 파일 삭제 전 반드시 백업
  - 가격 변경은 price_history.csv에도 기록
```

### 4단계: 도메인 지식 축적

Claude Code가 작업하면서 배운 것들을 기록해두기:

```
> 오늘 작업하면서 알게 된 것들을 CLAUDE.md에 추가해줘:
  orders/ 폴더는 월별로 아카이빙하고,
  취소된 주문은 cancelled/ 서브폴더로 이동함
```

## 온톨로지가 잘 작동하는지 테스트하기

새로운 Claude Code 세션을 시작하고:

```
> 이 프로젝트에 대해 알고 있는 것을 요약해줘
```

Claude Code가 CLAUDE.md를 읽고 프로젝트를 정확히 설명한다면 온톨로지가 잘 작동하는 거예요.

```
> 이 프로젝트에서 내가 실수로 삭제하면 안 되는 파일들이 뭐야?
```

위험 규칙을 정확히 알고 있다면 더욱 안심할 수 있어요.

## 온톨로지 발전시키기

온톨로지는 처음부터 완벽하게 만들 필요 없어요. 사용하면서 점점 쌓아가면 됩니다.

<StepByStep>
  <Step number={1}>
    **시작**: 기본 CLAUDE.md 만들기 (프로젝트 소개 + 폴더 설명)
  </Step>
  <Step number={2}>
    **확장**: 작업하면서 규칙과 주의사항 추가
  </Step>
  <Step number={3}>
    **심화**: 용어 사전, 에이전트 역할 정의(AGENTS.md) 추가
  </Step>
  <Step number={4}>
    **완성**: 도메인 전문 지식과 히스토리 축적
  </Step>
</StepByStep>

<Callout type="tip">
온톨로지를 잘 만들면 새 세션에서도 "처음 보는 프로젝트"가 아니라 "잘 아는 프로젝트"처럼 Claude Code가 작업해요. 한 번 투자해두면 이후 모든 작업이 훨씬 빠르고 정확해집니다.
</Callout>
