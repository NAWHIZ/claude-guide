# 오케스트레이션 — 여러 AI를 동시에

오케스트레이션은 여러 Claude Code 에이전트를 동시에, 또는 순서에 맞게 실행해서 복잡한 작업을 처리하는 방법이에요.

## 오케스트레이션이 뭔가요?

> **비유**: 오케스트라를 생각해보세요. 지휘자(여러분) 한 명이 바이올린, 첼로, 피아노 연주자들(각 에이전트)에게 각자의 파트를 맡겨요. 모두가 동시에 연주해서 하나의 완성된 음악이 만들어지죠. Claude Code 오케스트레이션도 마찬가지예요 — 여러 AI가 각자의 임무를 동시에 수행해서 혼자 했을 때보다 훨씬 빠르고 정확하게 작업을 완료해요.

## 왜 여러 에이전트를 쓰나요?

하나의 Claude Code로 모든 걸 처리하는 것보다 여러 에이전트를 쓰면:

- **빠름**: 독립적인 작업들을 동시에 처리할 수 있어요
- **전문화**: 각 에이전트가 특정 역할에 집중할 수 있어요
- **검토**: 한 에이전트가 작업하고 다른 에이전트가 검토하는 구조를 만들 수 있어요
- **규모**: 수십 개 파일을 병렬로 처리할 수 있어요

## 에이전트의 종류

Claude Code 생태계에서 쓰이는 에이전트 유형들이에요.

### 오케스트레이터 (지휘자)
전체 작업을 계획하고, 다른 에이전트들에게 세부 작업을 배분해요. 결과를 모아서 최종 검토도 합니다.

### 실행 에이전트 (연주자)
오케스트레이터의 지시에 따라 실제 작업을 수행해요. 파일 수정, 코드 작성, 분석 등을 담당해요.

### 검토 에이전트 (검수자)
다른 에이전트가 한 작업을 검토하고 품질을 확인해요.

## 실제 오케스트레이션 예시

### 예시 1: 대규모 문서 처리

100개 파일을 분석해서 보고서를 만들어야 할 때, 혼자 하면 오래 걸리지만 오케스트레이션을 사용하면:

```
> 이 폴더의 100개 파일을 10개씩 나눠서 10개의 에이전트가
  동시에 분석하고, 결과를 하나의 final_report.md로 합쳐줘
```

<BeforeAfter>
  <Before>
    에이전트 1개로 100개 파일 순차 처리
    → 예상 시간: 30분
  </Before>
  <After>
    에이전트 10개로 동시 처리 (각자 10개씩)
    → 예상 시간: 3~5분
  </After>
</BeforeAfter>

### 예시 2: 작성 → 검토 파이프라인

마케팅 문서를 작성하고 바로 검토받는 구조:

```
> 두 단계로 처리해줘:
  1단계: 에이전트 A가 제품 소개 문서 초안을 작성
  2단계: 에이전트 B가 초안을 검토하고 개선점을 제안
  최종: 개선된 버전을 final_doc.md로 저장
```

### 예시 3: 전문 역할 분담

복잡한 프로젝트를 역할별로 나눠서 처리:

```
> 신제품 런칭 준비를 세 팀에 나눠서 동시에 진행해줘:
  - 에이전트 1: 제품 소개 페이지 HTML 작성
  - 에이전트 2: 이메일 캠페인 문구 작성
  - 에이전트 3: SNS용 문구 5개 작성
  완료되면 모두 /launch 폴더에 저장해줘
```

## oh-my-claudecode 오케스트레이션 도구

Claude Code를 확장하는 oh-my-claudecode 플러그인을 설치했다면 더 강력한 오케스트레이션이 가능해요.

### `/team` — 팀 에이전트 실행

전문화된 에이전트 팀이 역할을 나눠서 작업해요:

```
> /team 이 프로젝트의 모든 문서를 검토하고 개선안을 제시해줘
```

사용 가능한 전문 에이전트들:
- `architect` — 구조 설계
- `executor` — 실제 구현
- `reviewer` — 코드/문서 검토
- `writer` — 문서 작성
- `security-reviewer` — 보안 검토

### `/ralph` — 완료될 때까지 반복 실행

목표가 달성될 때까지 Claude Code가 스스로 반복 작업하는 모드예요:

```
> /ralph 이 폴더의 모든 HTML 파일에서 깨진 링크를 찾아서 수정해줘.
  모든 링크가 정상이 될 때까지 계속해줘.
```

> **비유**: 일반 Claude Code는 "심부름꾼"이고, ralph 모드는 "담당 직원"이에요. 심부름꾼은 한 번 심부름을 가면 끝이지만, 담당 직원은 업무가 완료될 때까지 알아서 계속 처리해요.

### `/autopilot` — 전체 자동 실행

아이디어만 주면 분석 → 계획 → 실행 → 검토까지 알아서 해주는 모드예요:

```
> /autopilot 우리 회사 제품들의 한국어 소개 페이지를 만들어줘.
  products/ 폴더의 데이터를 기반으로.
```

<Callout type="tip">
오케스트레이션은 복잡한 작업에 효과적이에요. 단순한 작업(파일 하나 수정 등)은 그냥 일반 대화로 하는 게 더 빠를 수 있어요.
</Callout>

## 에이전트 실행 중 관리하기

여러 에이전트가 동시에 실행될 때 상황을 모니터링하고 필요하면 중단할 수 있어요.

**진행 상황 확인:**
```
> 현재 실행 중인 에이전트들의 상태를 알려줘
```

**특정 에이전트 중단:**
```
> /oh-my-claudecode:cancel
```

**작업 이어서 하기:**
에이전트가 중간에 중단됐어도 상태 파일이 남아 있어서 나중에 이어서 할 수 있어요.
