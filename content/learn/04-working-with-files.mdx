# 파일 읽고 수정 요청하기

Claude Code가 일반 AI 채팅과 다른 가장 큰 이유가 바로 이것이에요 — 파일을 직접 읽고 수정할 수 있다는 것. 이 기능을 제대로 활용하면 반복적인 파일 작업을 엄청나게 빠르게 처리할 수 있어요.

## Claude Code가 파일을 다루는 방식

Claude Code는 여러분이 요청하면 다음과 같은 순서로 파일 작업을 해요.

1. 파일을 찾아서 내용을 읽어요
2. 여러분의 요청을 이해해요
3. 수정이 필요하면 파일을 직접 고쳐요
4. 무엇을 했는지 설명해줘요

파일을 수정하기 전에 "이렇게 수정해도 될까요?"라고 먼저 물어볼 때도 있고, 바로 수정한 후 결과를 알려줄 때도 있어요. 중요한 파일을 다룰 때는 먼저 확인을 요청하는 게 좋아요.

## 파일 읽기 요청

파일 내용을 확인하고 싶을 때는 이렇게 요청해요.

```
> README.md 파일을 읽고 내용을 설명해줘
```

```
> config.json 파일 안에 어떤 설정이 있는지 알려줘
```

```
> 이 폴더에 있는 모든 .txt 파일을 읽고 어떤 내용인지 요약해줘
```

Claude Code가 해당 파일을 열어서 내용을 분석한 후 알기 쉽게 설명해줄 거예요. 기술적인 내용도 "이건 이런 의미예요"처럼 쉽게 풀어서 설명해달라고 추가로 요청할 수 있어요.

## 파일 수정 요청

파일을 수정하고 싶을 때는 어떤 파일의 어떤 내용을 어떻게 바꾸고 싶은지 명확히 말해주세요.

```
> index.html 파일에서 페이지 제목을 '내 웹사이트'로 바꿔줘
```

```
> data.csv 파일에서 '홍길동'이라는 이름을 모두 '김철수'로 바꿔줘
```

```
> style.css 파일의 배경색을 흰색에서 연한 회색(#f5f5f5)으로 바꿔줘
```

<Callout type="warning">
Claude Code는 파일을 실제로 수정해요. 한 번 수정된 파일은 원래대로 돌아가지 않을 수 있어요. 중요한 파일은 작업 전에 반드시 백업해두세요. 또는 Claude Code에게 "수정하기 전에 백업본을 먼저 만들어줘"라고 요청하는 방법도 있어요.
</Callout>

## 새 파일 만들기

새 파일을 생성하고 싶을 때는 파일 이름과 내용을 알려주세요.

```
> todo.txt 파일을 만들고 오늘 할 일 3가지를 써줘
```

```
> meeting-notes.md 파일을 만들어줘. 오늘 날짜로 회의 내용을 정리할 수 있는 템플릿 형식으로
```

```
> 간단한 index.html 파일을 만들어줘. '안녕하세요'라고 써있는 기본 웹페이지로
```

파일을 만들 때 내용까지 채워달라고 하거나, 빈 템플릿만 만들어달라고 할 수도 있어요.

<Callout type="tip">
"수정하기 전에 백업본 만들어줘"라고 요청하면 Claude Code가 원본 파일을 `파일명.backup` 형태로 복사해둔 후 수정을 진행해요. 혹시 결과가 마음에 들지 않으면 백업본으로 되돌릴 수 있어요.
</Callout>

## 여러 파일 한꺼번에 작업하기

여러 파일에 같은 작업을 해야 할 때도 한 번에 요청할 수 있어요.

```
> src 폴더 안의 모든 파일을 확인하고 어떤 파일인지 설명해줘
```

```
> 이 폴더에 있는 모든 HTML 파일에서 '구회사명'을 '새회사명'으로 바꿔줘
```

```
> 모든 .md 파일을 찾아서 각 파일의 첫 줄(제목)만 모아서 목록으로 보여줘
```

여러 파일을 다룰 때는 Claude Code가 작업 전에 무엇을 할 건지 먼저 설명해주는 경우가 많아요. 확인하고 진행하면 더 안전해요.

## 막연한 요청 vs 구체적인 요청

<BeforeAfter>
  <Before>
    **막연한 요청**: "파일 고쳐줘"

    Claude Code가 어떤 파일을, 어떻게 고쳐야 할지 알 수 없어서 되물어보거나 엉뚱한 작업을 할 수 있어요.
  </Before>
  <After>
    **구체적인 요청**: "src/index.html 파일에서 배경색을 흰색에서 연한 회색(#f5f5f5)으로 바꿔줘"

    어떤 파일(src/index.html)의 어떤 내용(배경색)을 어떻게(흰색→#f5f5f5) 바꿀지 명확해서 정확하게 작업할 수 있어요.
  </After>
</BeforeAfter>

## 작업 결과 확인하기

파일 수정 후에는 결과를 확인하는 습관을 들이세요.

```
> 방금 수정한 내용을 보여줘
```

```
> index.html 파일을 다시 읽어서 수정이 잘 됐는지 확인해줘
```

Claude Code가 수정한 내용을 보여주거나, 파일을 다시 읽어서 올바르게 변경됐는지 확인해줘요.
