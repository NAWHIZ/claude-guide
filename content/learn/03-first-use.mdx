# 처음 실행해보기

설치가 끝났다면 이제 직접 써볼 차례예요. 생각보다 훨씬 간단하답니다.

## Claude Code 실행하기

터미널을 열고 아래 명령어를 입력하세요.

```bash
claude
```

엔터를 누르면 Claude Code가 실행돼요. 처음 실행할 때는 몇 가지 안내 메시지가 나올 수 있어요. 그냥 엔터를 누르거나 안내에 따라 진행하면 됩니다.

## 인터페이스 살펴보기

Claude Code가 실행되면 이런 화면이 나타나요.

```
Claude Code v1.x.x
>
```

`>` 기호 옆에 커서가 깜빡이고 있어요. 여기에 하고 싶은 말을 입력하고 엔터를 누르면 됩니다. 일반 채팅과 똑같아요.

Claude가 응답을 작성하는 동안에는 텍스트가 화면에 실시간으로 나타나요. 응답이 끝나면 다시 `>` 프롬프트가 나타나고 여러분이 다음 내용을 입력할 수 있어요.

## 첫 번째 대화 해보기

먼저 간단하게 인사를 해볼까요?

```
> 안녕하세요! 저는 Claude Code를 처음 써봐요. 뭘 도와줄 수 있나요?
```

Claude Code가 자기소개와 함께 할 수 있는 일들을 설명해줄 거예요. 한국어로 물어보면 한국어로 대답합니다.

## 파일 목록 요청해보기

Claude Code의 핵심 기능은 파일을 다루는 것이에요. 현재 폴더의 파일들을 물어볼까요?

```
> 현재 폴더에 있는 파일들을 알려주세요
```

<Callout type="info">
Claude Code는 현재 실행 중인 폴더(디렉토리)의 파일을 볼 수 있어요. 터미널을 홈 폴더에서 열었다면 홈 폴더의 파일들을, Documents 폴더에서 열었다면 그 폴더의 파일들을 볼 수 있습니다.
</Callout>

Claude Code가 파일 목록을 읽고 어떤 파일들이 있는지 설명해줄 거예요.

## 종료하는 방법

대화를 끝내고 싶을 때는 두 가지 방법이 있어요.

- **방법 1**: `/exit` 또는 `/quit` 입력 후 엔터
- **방법 2**: `Ctrl+C` 키를 누르기 (Control 키와 C 키를 동시에)

## 막연한 질문 vs 구체적인 요청

Claude Code를 잘 쓰는 핵심 비결은 **구체적으로 말하는 것**이에요.

<BeforeAfter>
  <Before>
    "파일 좀 봐줘" — 어떤 파일인지, 무엇을 봐줘야 하는지 불명확해서 Claude가 뭘 해야 할지 모를 수 있어요.
  </Before>
  <After>
    "현재 폴더에 있는 README.md 파일을 읽고 이 프로젝트가 무엇인지 한국어로 설명해줘" — 어떤 파일을, 어떻게, 어떤 언어로 처리해야 하는지 명확해요.
  </After>
</BeforeAfter>

구체적으로 말할수록 Claude Code가 여러분이 원하는 것을 더 정확하게 해줍니다.

## 특정 폴더에서 시작하는 방법

Claude Code는 터미널이 현재 위치한 폴더를 기준으로 작동해요. 특정 프로젝트 폴더에서 시작하고 싶다면 먼저 그 폴더로 이동해야 해요.

`cd`는 "change directory"의 약자로, 폴더를 이동하는 명령어예요.

```bash
# Documents 폴더로 이동
cd Documents

# 그 안의 my-project 폴더로 이동
cd my-project

# Claude Code 실행
claude
```

Mac에서는 Finder에서 폴더를 열고 터미널에 `cd ` (cd 뒤에 공백 하나)를 입력한 다음, 폴더를 터미널 창으로 드래그 앤 드롭하면 자동으로 경로가 입력돼요.

## 실제로 이런 것들을 요청해보세요

처음에 어떻게 시작해야 할지 모르겠다면 아래 예시들을 그대로 복사해서 써보세요.

### 파일이 뭔지 파악할 때

```
> 이 폴더에 어떤 파일들이 있는지 설명해줘
```

```
> 이 프로젝트가 뭘 하는 건지 파일들을 보고 파악해줘
```

```
> README.md 파일 읽고 이게 뭔지 쉽게 설명해줘
```

### 파일 내용을 확인할 때

```
> config.json 파일 열어서 어떤 설정들이 있는지 알려줘
```

```
> 가장 최근에 수정된 파일이 어떤 건지 알려줘
```

### 수정 요청을 할 때

```
> index.html에서 '회사소개'를 '회사 소개'로 바꿔줘
```

```
> 이 폴더 안 .txt 파일들에서 '박부장'을 '박이사'로 모두 바꿔줘
```

### 새 파일을 만들 때

```
> 오늘 날짜로 회의록 파일 만들어줘. 제목, 참석자, 안건, 결정사항 항목이 있게.
```

```
> 이름, 이메일, 전화번호 컬럼으로 빈 CSV 파일 만들어줘
```

### 이해가 안 될 때

```
> 이 파일에서 이 부분이 무슨 뜻인지 쉬운 말로 설명해줘
```

```
> 이 에러 메시지가 왜 뜨는 건지, 어떻게 해야 하는지 알려줘
```

<Callout type="tip">
요청이 어색하게 느껴져도 괜찮아요. Claude Code는 문맥을 파악해서 의도를 이해해요. "파일 고쳐줘"처럼 짧게 말해도 되고, "이 파일에서 이 부분을 저렇게 바꿔주면 좋겠는데 어떻게 생각해?"처럼 대화하듯 말해도 됩니다.
</Callout>

처음에 어떻게 시작해야 할지 모르겠다면 이런 질문들로 시작해보세요.

```
> 이 폴더에 어떤 파일들이 있는지 설명해줘
```

```
> 이 프로젝트가 뭘 하는 프로젝트인지 파악해줘
```

```
> 여기서 내가 뭘 할 수 있을지 제안해줘
```

Claude Code가 폴더 내용을 살펴보고 상황에 맞는 답변을 해줄 거예요.
