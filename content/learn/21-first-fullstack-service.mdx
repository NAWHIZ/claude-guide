# 첫 번째 풀스택 서비스 만들기

지금까지 배운 것을 모두 합쳐서 실제로 동작하는 서비스를 만들어봐요. 이 가이드를 따라 하면 **이 클로드코드 가이드 사이트**와 동일한 방식으로 만들어진 서비스가 완성됩니다.

## 목표

**만들 것**: 팀 링크 모음 사이트
- 자주 쓰는 링크를 팀원들이 공유하는 사이트
- Supabase에 링크 저장
- Vercel로 배포
- 누구나 접근 가능한 URL 생성

**사용 스택**:
- **Next.js**: 웹사이트 뼈대
- **Supabase**: 링크 데이터 저장
- **Vercel**: 인터넷 공개
- **GitHub**: 코드 백업 + 자동 배포

> 코딩 한 줄 없이 Claude Code에게 말로만 시켜서 만들 거예요.

---

## 1단계: 프로젝트 폴더 준비

터미널을 열고 작업할 폴더로 이동해요.

```bash
cd Desktop
mkdir my-link-hub
cd my-link-hub
```

<Callout type="tip">
Mac에서 더 쉬운 방법: 바탕화면에 `my-link-hub` 폴더를 만들고, 터미널에 `cd ` 입력 후 폴더를 드래그앤드롭하면 경로가 자동 입력돼요.
</Callout>

## 2단계: Claude Code 실행 + 프로젝트 생성

```bash
claude
```

Claude Code가 실행되면:

```
> Next.js로 팀 링크 모음 사이트를 만들어줘.
  기능:
  - 링크 카드 목록 (제목, URL, 설명, 카테고리)
  - 카테고리별 필터
  - 깔끔한 모바일 반응형 디자인
  - Tailwind CSS 사용

  지금은 데이터는 임시로 하드코딩해줘.
  나중에 Supabase로 교체할 거야.
```

Claude Code가 Next.js 프로젝트를 생성하고 기본 페이지를 만들어줄 거예요.

**로컬에서 확인:**

```
> 개발 서버 실행해줘
```

브라우저에서 `http://localhost:3000` 접속해서 결과 확인.

## 3단계: GitHub 저장소 연결

[github.com](https://github.com)에서 새 저장소 만들기 (16번 아티클 참고)

```
> GitHub에 연결해줘.
  저장소: https://github.com/내계정명/my-link-hub.git
  초기 커밋하고 올려줘.
```

---

## 4단계: Supabase 연결

### Supabase 프로젝트 만들기

[supabase.com](https://supabase.com)에서 새 프로젝트 생성 (18번 아티클 참고)

### 테이블 만들기

Supabase SQL Editor에서:

```sql
CREATE TABLE links (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  description TEXT,
  category TEXT DEFAULT 'general',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE POLICY "Anyone can read"
ON links FOR SELECT USING (true);

CREATE POLICY "Anyone can insert"
ON links FOR INSERT WITH CHECK (true);

ALTER TABLE links ENABLE ROW LEVEL SECURITY;
```

### 테스트 데이터 넣기

```sql
INSERT INTO links (title, url, description, category) VALUES
('Anthropic 공식 사이트', 'https://anthropic.com', 'Claude 만든 회사', '참고자료'),
('Claude Code 문서', 'https://docs.anthropic.com/claude-code', 'Claude Code 공식 문서', '도구'),
('Supabase 문서', 'https://supabase.com/docs', '데이터베이스 연동 가이드', '도구');
```

### .env.local 설정

Supabase Settings → API에서 값 복사:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

```
> .env.local 파일에 Supabase 환경변수를 추가해줘:
  NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

### 코드에 Supabase 연결

```
> 하드코딩된 데이터를 Supabase links 테이블에서 실제로 가져오게 바꿔줘.
  @supabase/supabase-js 패키지를 설치하고
  환경변수를 사용해서 연결해줘.
```

로컬에서 다시 확인 (`http://localhost:3000`) — Supabase에서 데이터가 잘 오면 성공!

## 5단계: Vercel 배포

### Vercel 프로젝트 연결

1. [vercel.com](https://vercel.com) → **Add New Project**
2. GitHub 저장소 `my-link-hub` 선택
3. **Environment Variables** 추가:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. **Deploy** 클릭

약 2분 후 배포 완료. `my-link-hub.vercel.app` 주소로 접속 가능!

## 6단계: 기능 개선해보기

배포까지 됐으면 이제 기능을 더 추가해봐요. 모두 Claude Code에게 말로만 시키면 됩니다.

### 링크 추가 폼

```
> 사용자가 새 링크를 추가할 수 있는 폼을 만들어줘.
  제목, URL, 설명, 카테고리를 입력하면
  Supabase에 저장되고 목록이 새로고침돼.
```

### 검색 기능

```
> 링크 제목과 설명에서 검색할 수 있는 기능 추가해줘.
  실시간으로 필터링되면 좋겠어.
```

### 링크 삭제

```
> 각 링크 카드에 삭제 버튼 추가해줘.
  삭제하면 Supabase에서도 지워지게.
```

### 변경사항 배포

기능을 추가할 때마다:

```
> 변경 내용 GitHub에 올려줘. 커밋 메시지: "링크 추가 폼 구현"
```

GitHub에 올라가면 Vercel이 자동으로 새 버전을 배포해요.

## 전체 구조 이해하기

완성된 서비스의 흐름:

```
사용자가 브라우저에서 접속
       ↓
Vercel이 Next.js 앱 실행
       ↓
Next.js가 Supabase에서 링크 데이터 조회
       ↓
화면에 링크 카드 렌더링
       ↓
사용자가 새 링크 추가
       ↓
Supabase에 데이터 저장
       ↓
화면 자동 업데이트
```

이 구조가 이 클로드코드 가이드 사이트, 뉴스 자동 수집 시스템, AI 트렌드 위클리 모두 동일하게 적용된 패턴이에요.

## 막혔을 때 대처법

### 빌드 에러가 날 때

```
> 빌드 에러가 났어. 이 에러 해결해줘:
  [에러 메시지 전체 붙여넣기]
```

### Supabase 데이터가 안 올 때

```
> Supabase에서 데이터가 안 와. RLS 정책 확인하고 수정해줘.
```

### Vercel 배포가 실패할 때

```
> Vercel 배포 로그에서 이런 에러가 났어:
  [에러 메시지]
  고쳐줘.
```

<Callout type="tip">
막혔을 때 가장 효과적인 방법은 에러 메시지 전체를 그대로 Claude Code에게 붙여넣는 거예요. "이거 왜 안 돼?"보다 에러 메시지 붙여넣기가 훨씬 빠르게 해결돼요.
</Callout>
