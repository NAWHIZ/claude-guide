# 데이터 기반 의사결정: 지표 대시보드 만들기

"이번 주 DAU가 얼마예요?" "전환율이 어떻게 되죠?" — 이런 질문에 즉시 답하려면 데이터가 항상 눈앞에 있어야 해요. Claude Code로 PM 전용 지표 대시보드를 하루 안에 만들 수 있어요.

<Callout type="tip">
기술적인 구현은 Claude Code가 합니다. PM은 "어떤 지표가 필요한지"만 말하면 돼요.
</Callout>

---

## 어떤 대시보드를 만들건지 설계하기

Claude Code에게 요청하기 전에 필요한 지표 목록을 정리해요:

**핵심 지표 예시:**
- **성장**: DAU/MAU, 신규 가입자, 이탈률
- **참여도**: 세션 길이, 핵심 기능 사용률, 재방문율
- **전환**: 가입→활성화율, 무료→유료 전환율
- **수익**: MRR, ARPU, LTV

---

## 1단계: 데이터베이스 설계

```
> PM 지표 대시보드를 위한 Supabase 데이터베이스를 설계해줘.
  
  추적할 지표:
  - 일별 활성 사용자 수 (DAU)
  - 신규 가입자 수
  - 핵심 기능 사용 이벤트
  - 결제 전환 이벤트
  
  각 테이블의 SQL 생성 쿼리를 만들어줘.
  RLS 정책도 포함해줘 (읽기는 누구나, 쓰기는 서비스 키만).
```

---

## 2단계: 대시보드 앱 만들기

```
> Next.js로 PM 지표 대시보드를 만들어줘.
  
  페이지 구성:
  1. 오늘의 핵심 지표 (4개 숫자 카드)
     - DAU, 신규 가입, 전환, MRR
  2. 7일 트렌드 차트 (Line chart)
  3. 핵심 기능 사용률 (Bar chart)
  4. 코호트 리텐션 테이블
  
  데이터: Supabase에서 실시간 조회
  차트 라이브러리: recharts
  인증: 비밀번호 보호 (환경변수로 설정)
```

---

## 3단계: 데이터 파이프라인 만들기

대시보드에 보여줄 데이터를 어떻게 넣을까요:

**방법 1: CSV 임포트 (빠른 시작)**
```
> analytics-data.csv를 읽고
  Supabase daily_metrics 테이블에 데이터를 넣는
  스크립트를 만들어줘.
  
  CSV 컬럼: date, dau, new_users, conversions, mrr
  중복 날짜는 upsert로 처리해줘.
```

**방법 2: 이벤트 자동 수집 (실시간)**
```
> 웹 앱에서 사용자 이벤트를 자동으로 수집해서
  Supabase에 저장하는 코드를 만들어줘.
  
  추적할 이벤트:
  - page_view
  - button_click (이벤트 이름 포함)
  - feature_used (기능명 포함)
  - conversion (타입 포함)
```

---

## 4단계: 주간 리포트 자동 생성

```
> Supabase에서 지난 7일 데이터를 읽고
  주간 PM 리포트를 만들어줘.
  
  포함 항목:
  - 전주 대비 주요 지표 변화 (%)
  - 이번 주 하이라이트 (좋았던 것)
  - 우려사항 (나빠진 것)
  - 다음 주 주목할 포인트
  
  Markdown 형식으로 저장하고
  Notion에도 올려줘.
```

---

## 5단계: 이상 징후 알림 설정

```
> 다음 조건 중 하나라도 발생하면 Slack에 알림을 보내는
  모니터링 스크립트를 만들어줘.
  
  알림 조건:
  - DAU가 전일 대비 20% 이상 하락
  - 전환율이 3% 미만
  - 에러율이 1% 초과
  
  알림 채널: #product-alerts
  알림 메시지: 지표 이름, 현재값, 전일값, 변화율
```

<BeforeAfter>
  <Before>
    매주 직접 BigQuery/Mixpanel 접속 → 스크린샷 캡처 → Notion에 붙여넣기 → 2시간 소요
  </Before>
  <After>
    항상 켜져 있는 대시보드 → 이상 징후는 Slack 알림 → 주간 리포트 자동 생성 → 0분 소요
  </After>
</BeforeAfter>

## Vercel에 배포하기

```
> 대시보드를 Vercel에 배포해줘.
  환경변수:
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY
  - DASHBOARD_PASSWORD
```

GitHub에 올라가면 Vercel이 자동으로 배포해요. 팀원들과 URL을 공유하면 모두가 같은 데이터를 볼 수 있어요.
