# 프로젝트에서 활용하기

Claude Code는 특정 프로젝트 폴더에서 실행하면 훨씬 더 강력해져요. 폴더 안의 모든 파일을 파악하고, 전체 맥락을 이해한 상태에서 도움을 줄 수 있거든요.

## 프로젝트란 무엇인가요?

Claude Code에서 "프로젝트"는 하나의 폴더예요. 예를 들어:

- 블로그 글을 모아둔 폴더
- 회사 보고서 파일들이 있는 폴더
- 개인 프로젝트 코드가 있는 폴더

Claude Code를 특정 폴더에서 실행하면, 그 폴더가 작업 공간이 됩니다.

## 프로젝트 폴더에서 시작하기

작업할 폴더로 먼저 이동한 다음 Claude Code를 실행해요.

```bash
# 먼저 폴더로 이동
cd /Users/사용자이름/Documents/내프로젝트

# 그 다음 Claude Code 실행
claude
```

이렇게 하면 Claude Code가 그 폴더의 파일들을 바로 볼 수 있어요.

<Callout type="tip">
Mac Finder에서 폴더를 열어놓은 상태라면, 터미널에 `cd ` (공백 포함)를 입력한 다음 Finder에서 폴더를 드래그&드롭하면 경로가 자동으로 입력돼요.
</Callout>

## 프로젝트 파악시키기

새 프로젝트 폴더에서 처음 시작할 때는 Claude에게 전체 파악을 요청해보세요:

```
> 이 프로젝트 폴더를 살펴보고 어떤 내용인지 설명해줘
```

Claude가 폴더 구조와 파일들을 읽어서 전체 맥락을 파악해요. 그 다음 작업들이 훨씬 정확해집니다.

## CLAUDE.md — 프로젝트 설명서

`CLAUDE.md`는 프로젝트 폴더에 만들어두는 특별한 파일이에요. Claude Code를 실행할 때마다 이 파일을 자동으로 읽어서 프로젝트 맥락을 기억해요.

쉽게 말하면 **"Claude Code에게 우리 프로젝트를 설명해두는 메모**"예요.

### CLAUDE.md 만들기

```
> 이 프로젝트를 위한 CLAUDE.md 파일을 만들어줘. 이 프로젝트는 회사 마케팅 자료를 관리하는 폴더야. 주요 폴더로는 images/, documents/, templates/이 있어.
```

또는 직접 만들어도 돼요:

```
> CLAUDE.md 파일을 만들어줘. 내용은 이렇게:
> - 이 폴더는 2024년 신제품 런칭 프로젝트
> - 주요 파일: brief.md, timeline.xlsx, assets 폴더
> - 이미지는 모두 1920x1080 해상도로 맞춰야 함
> - 파일 이름은 영어 소문자와 하이픈만 사용
```

### CLAUDE.md에 넣으면 좋은 내용

- 프로젝트 목적과 배경
- 중요한 파일/폴더 설명
- 따라야 할 규칙이나 형식
- 자주 사용하는 용어 설명
- 주의해야 할 사항

## 여러 파일에 걸친 작업

프로젝트 맥락이 잡히면 여러 파일에 걸친 복잡한 작업도 가능해요:

```
> 이 프로젝트의 모든 마케팅 문서에서 2023년을 2024년으로 바꿔줘
```

```
> 회의록 폴더의 모든 파일을 읽고 결정 사항만 모아서 decisions.md 파일로 정리해줘
```

```
> 이미지 폴더에 있는 파일들의 목록을 README.md에 추가해줘
```

## 실전 프로젝트 예시

**상황**: 팀 블로그 관리

```
> 이 블로그 폴더를 살펴보고 각 카테고리별로 글이 몇 개인지 세어줘
```

```
> posts 폴더의 모든 마크다운 파일에서 published: false인 글 목록을 보여줘
```

```
> 가장 최근에 수정된 글 5개는 뭐야?
```

이처럼 프로젝트 맥락이 있으면 훨씬 구체적이고 정확한 작업이 가능해요.
