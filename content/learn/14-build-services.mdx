# MCP로 실제 서비스 만들기

Claude Code + MCP 조합이면 실제로 사람들이 쓸 수 있는 서비스를 만들 수 있어요. 이 페이지에서는 코딩 없이 구현 가능한 4가지 실전 청사진을 소개할게요.

<Callout type="tip">
이 사이트(claude-guide) 자체가 Claude Code만으로 만들어진 서비스예요. 13개 학습 페이지, 인터랙티브 퀴즈, Vercel 배포까지 — 작성자의 코딩 지식은 0이었어요.
</Callout>

---

## 청사진 1: 학습/정보 사이트

**이 사이트처럼 만들기**

### 필요한 것

| 도구 | 역할 | 비용 |
|------|------|------|
| Claude Code | 전체 제작 지휘 | 유료 (월 $20) |
| Next.js | 웹사이트 뼈대 | 무료 |
| Vercel | 인터넷에 공개 | 무료 |
| Supabase MCP | 사용자 데이터 저장 | 무료 (소규모) |
| Tavily MCP | 최신 정보 검색 | 무료 플랜 있음 |

### Claude Code에게 이렇게 말하면 돼요

```
> Next.js로 [주제] 학습 사이트 만들어줘.
  - 페이지마다 퀴즈 포함
  - 모바일에서도 잘 보이게
  - Vercel에 배포까지
```

```
> Supabase 연결해서 퀴즈 점수를 저장하게 해줘
```

```
> Tavily MCP로 최신 뉴스를 매일 자동으로 가져와서
  "오늘의 업데이트" 섹션에 넣어줘
```

### 이런 사이트를 만들 수 있어요

- 업계 트렌드 뉴스레터 사이트
- 사내 교육 플랫폼
- 제품 사용 가이드 사이트
- 스터디 그룹 자료 공유 사이트

<Callout type="info">
**소요 시간**: Claude Code와 함께 하면 하루~이틀 안에 MVP(최소 기능 버전) 완성 가능해요.
</Callout>

## 청사진 2: 업무 자동화 봇

**매일 반복하는 일을 자동으로**

### 필요한 것

| 도구 | 역할 | 비용 |
|------|------|------|
| Slack MCP | 메시지 읽기/보내기 | Slack 유료 플랜 필요 |
| Notion MCP | 문서 읽기/쓰기 | Notion 무료 플랜 가능 |
| Tavily MCP | 웹에서 정보 수집 | 무료 플랜 있음 |

### 자동화할 수 있는 것들

**📊 일일 리포트 자동 생성**
```
> 매일 오전 9시에:
  1. Notion "작업 현황" 데이터베이스를 읽어서
  2. 완료/진행중/대기 항목을 정리하고
  3. Slack #daily-report 채널에 요약을 보내줘
```

**📰 경쟁사 모니터링**
```
> 매주 월요일에:
  1. Tavily로 [경쟁사 이름] 최신 소식 검색해서
  2. Notion "경쟁사 동향" 페이지에 추가해줘
  3. 중요한 내용이 있으면 Slack으로 알려줘
```

**📝 회의 후 자동 정리**
```
> meeting_notes.txt 파일을 읽고:
  1. 핵심 결정 사항 정리
  2. 담당자별 할일 목록 추출
  3. Notion 회의록 데이터베이스에 저장
  4. 담당자들에게 Slack DM으로 할일 목록 발송
```

<BeforeAfter>
  <Before>
    매일 Notion 확인 → 내용 복사 → Slack에 수동 게시 → 30분 소요
  </Before>
  <After>
    Claude Code + MCP 자동화 → 0분 소요, 매일 자동 실행
  </After>
</BeforeAfter>

## 청사진 3: 데이터 분석 대시보드

**숫자로 상황을 파악하는 도구**

### 필요한 것

| 도구 | 역할 | 비용 |
|------|------|------|
| Supabase MCP | 데이터 저장/조회 | 무료 (소규모) |
| Next.js | 대시보드 화면 | 무료 |
| Vercel | 배포 | 무료 |

### Claude Code에게 이렇게 말하면 돼요

```
> Supabase에 있는 sales 테이블을 읽어서
  월별 매출 추이를 보여주는 차트 페이지를 만들어줘
```

```
> customers 테이블에서:
  - 이번 달 신규 가입자 수
  - 지역별 분포
  - 재구매율
  을 한눈에 볼 수 있는 대시보드 만들어줘
```

```
> 매출 데이터 CSV 파일을 Supabase에 업로드하고
  자동으로 그래프로 시각화해줘
```

### 실제 구현 예시

이런 대시보드가 하루 안에 만들어져요:

```
> 우리 쇼핑몰 주문 데이터(orders.csv)를 분석해서:
  1. 오늘 주문 수 / 매출
  2. 인기 상품 Top 5
  3. 시간대별 주문 패턴
  을 보여주는 관리자 페이지 만들어줘.
  비밀번호로 접근 제한도 걸어줘.
```

<Callout type="tip">
CSV 파일이 있으면 Supabase MCP로 바로 업로드하고, Claude Code가 자동으로 테이블 구조를 잡아줘요. 데이터베이스 지식 없어도 됩니다.
</Callout>

## 청사진 4: PDF 자동 생성 시스템

**문서를 자동으로 만들고 보내기**

### 필요한 것

| 도구 | 역할 | 비용 |
|------|------|------|
| Claude Code | 문서 내용 생성 | 유료 |
| Chrome Headless | HTML → PDF 변환 | 무료 (내장) |
| Slack MCP | 완성된 PDF 공유 | 선택 |

### 사용 방법

**이력서/포트폴리오 PDF**
```
> resume_data.md 파일을 읽고
  전문적인 이력서 HTML을 만든 다음
  PDF로 변환해서 Desktop에 저장해줘
```

**견적서/계약서 자동 생성**
```
> quote_template.html을 기반으로
  client_info.csv의 각 고객별 견적서를 만들어줘.
  각 파일 이름은 "견적서_[고객사명]_[날짜].pdf"로
```

**월간 보고서 자동화**
```
> Supabase에서 이번 달 데이터를 읽고
  경영진 보고서 PDF를 만들어줘.
  차트와 표 포함해서 전문적으로.
```

<Callout type="info">
Mac에는 Chrome이 설치되어 있으면 PDF 변환이 바로 돼요. 별도 소프트웨어 설치가 필요 없어요.
</Callout>

## MCP 연결 비용 총정리

실제로 돈이 얼마나 드는지 정리해볼게요:

| 서비스 | 무료 한도 | 초과 시 |
|--------|-----------|---------|
| **Vercel** (배포) | 무료 플랜으로 웬만한 사이트 가능 | 월 $20부터 |
| **Supabase** (DB) | DB 500MB, 월 5만 행 읽기 | 월 $25부터 |
| **Notion MCP** | Notion 무료 플랜 사용 가능 | Notion 유료 플랜 필요 시 |
| **Slack MCP** | Slack 무료 플랜 제한적 | Slack 유료 플랜 |
| **Tavily** (검색) | 월 1,000회 무료 | 월 $20부터 |
| **Figma MCP** | Figma 무료 플랜 가능 | Figma 유료 시 |

**현실적인 스타트업/소규모 사업자 비용**:
- Claude Code: 월 $20 (필수)
- Vercel + Supabase: 무료 플랜으로 시작 가능
- 나머지: 필요한 것만 추가

**→ 월 $20으로 꽤 많은 걸 만들 수 있어요.**

## 지금 바로 시작하는 순서

처음이라면 이 순서를 추천해요:

### 1단계: 가장 간단한 것부터
```
> 내 이력서를 HTML로 예쁘게 만들고 PDF로 변환해줘
```
→ 코딩 없이 결과물이 나오는 경험을 먼저 해보세요.

### 2단계: 웹사이트 배포 경험
```
> 간단한 포트폴리오/소개 페이지를 만들고 Vercel에 배포해줘
```
→ 내 주소(도메인)를 가진 사이트가 생겨요.

### 3단계: 데이터 연결
```
> Supabase 연결해서 방문자가 메모를 남길 수 있게 해줘
```
→ 사용자가 쓰는 진짜 서비스가 돼요.

### 4단계: 자동화 추가
```
> 새 메모가 올라오면 내 Slack으로 알림 보내줘
```
→ 알아서 돌아가는 시스템이 완성돼요.

<Callout type="tip">
각 단계에서 Claude Code에게 "이게 어떻게 동작하는 거야?"라고 물어보면서 배워가세요. 만들면서 자연스럽게 이해가 돼요.
</Callout>
