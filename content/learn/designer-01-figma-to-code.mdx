# Figma 디자인을 코드로 자동 변환하기

"제 디자인이 이렇게 구현됐어요"라는 말을 더 이상 듣지 않아도 돼요. Figma MCP를 연결하면 Claude Code가 직접 내 Figma 파일을 읽고 코드로 변환합니다.

<Callout type="tip">
이 방법으로 만들어진 컴포넌트는 100% 일치하지 않을 수 있어요. 하지만 개발자와의 소통 시간을 80% 줄이고, 디자이너가 직접 결과물을 확인하며 수정할 수 있어요.
</Callout>

---

## 필요한 것

- Figma MCP 설정 완료 ([MCP 설정 가이드 →](/learn/13-mcp-servers))
- Figma 파일 URL 또는 노드 ID
- Next.js 프로젝트 (없으면 Claude Code가 생성)

---

## 1단계: Figma에서 컴포넌트 URL 가져오기

1. Figma에서 변환하고 싶은 컴포넌트/화면 선택
2. 오른쪽 클릭 → "Copy link to selection"
3. URL 복사 (예: `https://www.figma.com/design/XXXXX/...?node-id=123:456`)

---

## 2단계: 단일 컴포넌트 변환

```
> 이 Figma 컴포넌트를 React 컴포넌트로 변환해줘:
  https://www.figma.com/design/XXXXX/my-design?node-id=123:456
  
  요구사항:
  - TypeScript + Tailwind CSS
  - props로 커스터마이즈 가능하게
  - 모바일 반응형
  - 파일: components/ui/[컴포넌트명].tsx
```

---

## 3단계: 전체 화면 변환

단일 컴포넌트가 아닌 전체 페이지도 변환할 수 있어요:

```
> 이 Figma 화면을 Next.js 페이지로 변환해줘:
  https://www.figma.com/design/XXXXX/my-design?node-id=1:2
  
  - 각 섹션을 별도 컴포넌트로 분리
  - components/sections/ 폴더에 저장
  - app/page.tsx에서 조합
  - 하드코딩된 텍스트는 props로 추출
```

---

## 4단계: 디자인 시스템 추출

Figma의 Design Token을 코드로 가져올 수 있어요:

```
> 이 Figma 파일의 컬러 팔레트와 타이포그래피를
  Tailwind 설정으로 변환해줘:
  https://www.figma.com/design/XXXXX/design-system
  
  출력: tailwind.config.ts의 theme.extend 섹션
  
  포함:
  - 브랜드 색상 (Primary, Secondary, Neutral)
  - 폰트 크기 스케일
  - 간격 값
```

---

## 5단계: 구현 결과 확인 및 수정

변환된 코드를 실행해서 결과를 확인해요:

```
> 개발 서버를 실행해서 결과를 보여줘.
  http://localhost:3000에서 확인할게.
```

원하는 대로 수정하려면:

```
> 버튼 크기가 Figma보다 작아. 
  padding을 px-6 py-3으로 늘려줘.
  
  그리고 hover 효과도 Figma 프로토타입처럼
  배경색이 더 어두워지게 해줘.
```

---

## Figma → 코드 자동화 파이프라인

디자인 업데이트가 자주 있다면 파이프라인을 만들어요:

```
> Figma 파일이 업데이트될 때마다
  자동으로 코드를 업데이트하는 스크립트를 만들어줘.
  
  방식: Figma Webhook → Claude Code 자동 실행
  대상 컴포넌트: components/figma/ 폴더
  
  단, 기존 파일을 덮어쓰기 전에 diff를 보여줘.
```

<Callout type="info">
Figma → 코드 자동화는 디자이너-개발자 협업 방식을 근본적으로 바꾸고 있어요. 2025년 기준 많은 팀이 이 방식으로 핸드오프 시간을 70% 단축했어요.
</Callout>

---

## 실전 팁: 좋은 변환을 위한 Figma 준비

Claude Code가 더 정확하게 변환하려면:

1. **컴포넌트명 규칙 설정**: Figma에서 컴포넌트 이름을 명확하게 (Button/Primary, Card/Default)
2. **Auto Layout 사용**: Flexbox와 1:1 매핑되어 코드가 더 정확해요
3. **Color Style 등록**: Figma Color Styles가 있으면 CSS 변수로 잘 변환돼요
4. **텍스트 Style 등록**: Typography Style이 있으면 Tailwind 클래스와 매핑
