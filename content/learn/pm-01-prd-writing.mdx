# AI로 PRD 10분 만에 작성하기

PRD(Product Requirements Document) 작성에 하루를 쏟던 시절은 끝났어요. Claude Code와 함께라면 아이디어를 설명하는 것만으로 완성도 높은 PRD 초안이 나옵니다.

<Callout type="tip">
이 방법을 쓰면 PRD 초안 작성 시간이 평균 3시간 → 15분으로 줄어요. 남은 시간은 팀과 논의하고 다듬는 데 쓰세요.
</Callout>

---

## 왜 AI로 PRD를 쓰나요?

PRD의 가장 힘든 부분은 **빈 화면을 채우는 것**이에요. 머릿속엔 아이디어가 있는데, 그걸 구조화된 문서로 만드는 과정에서 막히죠. Claude Code는 이 부분을 대신해줄 수 있어요.

- 아이디어 → 구조화된 문서로 변환
- 놓친 요구사항을 질문으로 발굴
- 기존 PRD를 템플릿 삼아 일관된 형식 유지
- 기술 용어와 비즈니스 언어 사이 번역

---

## 1단계: 기본 PRD 초안 만들기

Claude Code 터미널에서:

```
> 다음 아이디어로 PRD 초안을 만들어줘.

기능명: 팀 링크 공유 허브
배경: 팀원들이 자주 쓰는 링크를 매번 Slack에 물어보는 상황
목표: 팀 링크를 한 곳에 모아서 누구나 쉽게 찾을 수 있게

PRD 포함 항목:
- 배경 및 목적
- 사용자 스토리 (최소 5개)
- 기능 요구사항 (Must/Should/Could)
- 비기능 요구사항
- 성공 지표
- 제외 범위

파일명: prd-link-hub.md로 저장해줘.
```

---

## 2단계: 사용자 스토리 강화하기

초안이 나오면 사용자 스토리를 더 구체화해요:

```
> prd-link-hub.md의 사용자 스토리를 강화해줘.
  
  각 스토리마다:
  - As a [역할], I want [기능], So that [가치]
  - 수락 기준 (Acceptance Criteria) 3개 이상
  - 엣지 케이스 1-2개
  
  타겟 사용자는 5-20명 규모 스타트업 팀이야.
```

---

## 3단계: 경쟁사 분석 섹션 추가

```
> prd-link-hub.md에 경쟁사 분석 섹션을 추가해줘.
  
  비교 대상:
  - Notion (링크 관리)
  - Confluence (팀 위키)
  - Linktree (링크 모음)
  
  비교 항목: 기능, 가격, 우리가 더 잘할 수 있는 포인트
```

---

## 4단계: 개발팀을 위한 기술 스펙 추가

```
> prd-link-hub.md에 기술 구현 참고사항을 추가해줘.
  
  개발 스택: Next.js + Supabase + Vercel
  
  추가할 내용:
  - API 엔드포인트 목록 (RESTful)
  - 데이터베이스 스키마 초안
  - 예상 성능 요구사항 (응답시간, 동시 접속)
```

---

## 5단계: Notion에 자동 저장 (선택)

Notion MCP가 연결되어 있다면:

```
> prd-link-hub.md 내용을 Notion "제품 문서" 데이터베이스에
  새 페이지로 저장해줘.
  
  속성:
  - 상태: 초안
  - 담당자: 나
  - 마감일: 이번 주 금요일
```

---

## 실전 팁: PRD 리뷰 요청하기

작성한 PRD를 Claude Code에게 검토받을 수도 있어요:

```
> prd-link-hub.md를 PM 관점에서 리뷰해줘.
  
  확인 포인트:
  - 모호한 요구사항이 있나?
  - 빠진 엣지 케이스는?
  - 성공 지표가 측정 가능한가?
  - 개발팀이 헷갈릴 부분은?
```

<Callout type="info">
Claude Code가 지적한 부분이 실제로 스프린트 중 문제가 되는 경우가 많아요. 리뷰 한 번에 스프린트 중 발생하는 질문을 60% 이상 줄일 수 있어요.
</Callout>

---

## PRD 템플릿 라이브러리 만들기

여러 번 쓰다 보면 회사만의 PRD 패턴이 생겨요. 이걸 템플릿으로 저장해두면:

```
> 지금까지 만든 PRD 패턴을 기반으로
  우리 팀 PRD 템플릿을 만들어줘.
  
  templates/prd-template.md로 저장.
  변수는 [대괄호]로 표시해줘.
```
