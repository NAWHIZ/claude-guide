# Vercel 배포 실전

코드가 완성됐으면 이제 인터넷에 공개할 차례예요. Vercel은 Next.js 서비스를 가장 쉽게 배포할 수 있는 플랫폼이에요.

## Vercel이 뭔가요?

Vercel은 웹 서비스를 인터넷에 올려주는 호스팅 플랫폼이에요.

> **비유**: 내가 만든 간판(웹사이트)을 실제 도로변(인터넷)에 설치해주는 서비스예요. 간판을 아무리 잘 만들어도 가게 안에만 있으면 아무도 못 보죠. Vercel이 그 간판을 전 세계 누구나 볼 수 있게 설치해줍니다.

Vercel의 장점:
- Next.js와 완벽 호환 (같은 회사가 만듦)
- GitHub와 연동하면 코드 올릴 때마다 자동 배포
- 무료 플랜으로 웬만한 서비스 운영 가능
- `내프로젝트.vercel.app` 주소 자동 제공

## 1단계: Vercel 계정 만들기

1. [vercel.com](https://vercel.com) 접속
2. **Sign Up** 클릭
3. **GitHub으로 계속하기** 선택 (GitHub 계정 연동이 편해요)
4. Vercel이 GitHub 접근 권한 요청 → 허용

## 2단계: 프로젝트 가져오기

GitHub에 코드가 올라가 있다면:

1. Vercel 대시보드에서 **Add New → Project**
2. **Import Git Repository** 선택
3. 배포할 GitHub 저장소 선택
4. **Import** 클릭

Next.js 프로젝트라면 Vercel이 자동으로 감지해서 설정해줘요.

## 3단계: 환경변수 설정

배포 전에 환경변수를 먼저 등록해야 해요. `.env.local` 파일은 로컬에만 있고 GitHub엔 올라가지 않기 때문입니다.

**Environment Variables** 섹션에서:

```
ANTHROPIC_API_KEY = sk-ant-api03-...
NEXT_PUBLIC_SUPABASE_URL = https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJhbGci...
CRON_SECRET = 나만아는비밀문자열
```

각 항목을 하나씩 추가하고 **Save** 클릭.

<Callout type="warning">
환경변수를 빠뜨리면 배포는 성공해도 서비스가 동작하지 않아요. .env.local의 모든 항목을 Vercel에도 동일하게 등록해주세요.
</Callout>

## 4단계: 배포

**Deploy** 버튼을 클릭하면 배포가 시작돼요.

배포 과정:
1. Vercel이 GitHub에서 코드를 가져옴
2. `npm run build` 실행 (약 1~3분)
3. 배포 완료 → URL 생성

배포가 완료되면 `프로젝트명.vercel.app` 주소로 접속할 수 있어요.

## 배포 후 확인할 것들

### 로그 보기

뭔가 잘못됐을 때 원인 파악:

1. Vercel 대시보드 → **Deployments** 탭
2. 해당 배포 클릭 → **Build Logs** 확인

빌드 에러 메시지가 나오면 Claude Code에게:

```
> Vercel 빌드 로그에서 이런 에러가 났어:
  [에러 메시지 붙여넣기]
  어떻게 고치면 돼?
```

### 함수 로그 (실시간 에러 확인)

서비스가 실행 중 에러날 때:

1. Vercel 대시보드 → **Logs** 탭
2. 실시간으로 서버 에러 확인 가능

## 자동 배포 흐름 이해하기

GitHub와 연동 후 작업 흐름:

```
내가 Claude Code로 기능 추가
    ↓
"GitHub에 올려줘" 요청
    ↓
Claude Code가 commit + push
    ↓
Vercel이 자동 감지
    ↓
자동 빌드 & 배포 (1~3분)
    ↓
새 버전 공개
```

매번 수동으로 배포할 필요 없이, GitHub에 올리는 것만으로 끝이에요.

## 크론(Cron) 설정하기

매일 자동 실행되는 기능(예: 뉴스 자동 수집)을 설정하려면 `vercel.json` 파일이 필요해요.

```
> vercel.json 파일 만들어줘.
  매일 오전 10시(한국 시간)에 /api/daily-update를 실행하는 크론을 추가해줘.
```

Claude Code가 만들어주는 파일:

```json
{
  "crons": [
    {
      "path": "/api/daily-update",
      "schedule": "0 1 * * *"
    }
  ]
}
```

<Callout type="info">
크론 시간은 UTC 기준이에요. 한국 시간(KST)은 UTC+9이므로, 한국 오전 10시 = UTC 오전 1시 → `0 1 * * *`
</Callout>

## 커스텀 도메인 연결하기

`내프로젝트.vercel.app` 말고 직접 구매한 도메인을 쓰고 싶다면:

1. Vercel 프로젝트 → **Settings → Domains**
2. 구매한 도메인 입력 (예: `myservice.com`)
3. Vercel이 알려주는 DNS 설정을 도메인 구매처에서 적용

도메인은 가비아, 후이즈, Namecheap 등에서 구매할 수 있어요.

## 무료 플랜 한도

Vercel 무료 플랜(Hobby)으로 할 수 있는 것:

| 항목 | 한도 |
|------|------|
| 배포 | 무제한 |
| 대역폭 | 월 100GB |
| 서버리스 함수 | 월 100시간 |
| 크론 작업 | 하루 1개 |
| 팀원 | 1명 (혼자) |

소규모 서비스는 무료 플랜으로 충분해요.
