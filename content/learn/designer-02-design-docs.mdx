# 디자인 스펙 문서, 자동으로 만들기

디자인을 완성한 후 가장 시간이 걸리는 작업 중 하나가 스펙 문서 작성이에요. 여백, 폰트 크기, 색상 코드, 인터랙션 설명... Claude Code로 이 과정을 자동화할 수 있어요.

---

## 어떤 문서를 자동화할 수 있나요?

| 문서 유형 | 내용 | 대상 |
|---------|------|------|
| 컴포넌트 스펙 | Props, 상태, 사용법 | 개발팀 |
| Redline 문서 | 크기, 여백, 색상 | 개발팀 |
| 인터랙션 가이드 | 애니메이션, 트랜지션 | 개발팀 |
| 접근성 체크리스트 | WCAG 기준 | QA팀 |

---

## 1단계: Figma에서 컴포넌트 정보 추출

```
> 이 Figma 컴포넌트를 읽고 스펙 문서를 만들어줘:
  https://www.figma.com/design/XXXXX?node-id=123:456
  
  스펙 문서에 포함:
  - 컴포넌트 이름 및 설명
  - Props 목록 (이름, 타입, 기본값, 설명)
  - 상태 (default, hover, active, disabled, focus)
  - 크기 및 여백 (px 단위)
  - 색상 (HEX 코드 + 의미)
  - 사용 예시 코드
  
  파일: docs/components/Button.md로 저장.
```

---

## 2단계: 전체 디자인 시스템 문서화

```
> 이 Figma 파일의 모든 컴포넌트를 읽고
  디자인 시스템 문서를 만들어줘:
  https://www.figma.com/design/XXXXX/design-system
  
  docs/design-system/ 폴더에 컴포넌트별 파일로 저장.
  
  목차 파일(README.md)도 만들어줘.
```

---

## 3단계: 개발자 핸드오프 문서

```
> Button 컴포넌트 스펙을 기반으로
  개발자 핸드오프 문서를 만들어줘.
  
  포함 항목:
  1. 구현 우선순위 (MVP vs 나중에)
  2. 기술 구현 노트 (주의사항)
  3. 엣지 케이스 (긴 텍스트, 아이콘 없을 때 등)
  4. 브라우저 지원 범위
  5. 접근성 요구사항 (aria, focus trap 등)
```

---

## 4단계: 접근성 체크리스트 자동 생성

2025년 이후로 접근성은 필수예요:

```
> 내 컴포넌트 목록을 보고
  WCAG 2.1 AA 기준 접근성 체크리스트를 만들어줘.
  
  컴포넌트: Button, Input, Modal, Dropdown, Card
  
  각 컴포넌트별:
  - 색상 대비 기준 (4.5:1 이상)
  - 키보드 접근성
  - 스크린 리더 지원
  - 포커스 표시
  
  체크리스트 형식 (- [ ] 항목)으로 만들어줘.
```

---

## 5단계: 변경사항 자동 기록

디자인이 업데이트될 때마다 변경 이력을 남겨요:

```
> 어제 날짜의 디자인 스펙과 오늘 날짜를 비교해서
  변경사항을 정리해줘.
  
  형식:
  ## [날짜] 업데이트
  ### 추가된 것
  ### 변경된 것
  ### 제거된 것
  ### 개발팀에게 영향을 주는 변경사항 (★ 표시)
  
  docs/CHANGELOG.md에 맨 위에 추가해줘.
```

---

## 인터랙션 가이드 작성하기

애니메이션과 트랜지션 스펙은 가장 빠뜨리기 쉬운 부분이에요:

```
> 다음 인터랙션들의 스펙을 정리해줘:
  
  - 버튼 hover: 배경색 전환
  - 모달 열기/닫기: 페이드 + 슬라이드
  - 토스트 알림: 슬라이드 인 + 자동 사라짐
  - 드롭다운: 펼치기/접기
  
  각 인터랙션별:
  - 시작 상태 → 끝 상태
  - duration (ms)
  - easing 함수
  - CSS 코드 (transition/animation)
```

<Callout type="info">
좋은 인터랙션 스펙이 있으면 개발자가 "이거 어떻게 구현하면 돼요?" 질문을 70% 줄일 수 있어요.
</Callout>

---

## Notion에 문서 자동 배포

```
> docs/design-system/ 폴더의 모든 문서를
  Notion "디자인 시스템" 페이지 하위에 업로드해줘.
  
  각 컴포넌트 문서는 별도 페이지로.
  목차 페이지도 자동 생성해줘.
```
