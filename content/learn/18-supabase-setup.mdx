# Supabase 시작하기

사용자 데이터를 저장하고, 실시간으로 읽어오고, 여러 사람이 함께 쓸 수 있는 서비스를 만들려면 **데이터베이스**가 필요해요. Supabase는 개발자가 아니어도 쉽게 쓸 수 있는 데이터베이스 서비스예요.

## Supabase가 뭔가요?

Supabase는 데이터를 저장하고 관리하는 온라인 데이터베이스 서비스예요.

> **비유**: 엑셀 스프레드시트를 상상해보세요. 행과 열로 데이터를 저장하는 것처럼, Supabase도 테이블 형태로 데이터를 저장해요. 차이점은 인터넷에 연결되어 있어서 여러분의 웹 서비스와 실시간으로 데이터를 주고받을 수 있다는 거예요.

Supabase로 할 수 있는 것:
- 사용자가 입력한 데이터 저장 (뉴스 기사, 댓글, 회원 정보 등)
- 저장된 데이터를 화면에 보여주기
- 조건에 맞는 데이터만 골라서 가져오기
- 실시간 업데이트

## 1단계: Supabase 계정 만들기

1. [supabase.com](https://supabase.com) 접속
2. **Start your project** 클릭
3. GitHub 계정으로 로그인 (편리해요)

## 2단계: 새 프로젝트 만들기

1. 대시보드에서 **New Project** 클릭
2. Organization 선택 (처음이면 기본 조직 사용)
3. 프로젝트 이름 입력 (예: `my-service-db`)
4. 데이터베이스 비밀번호 설정 (나중에 필요하니 저장해두세요)
5. Region: **Northeast Asia (Seoul)** 선택
6. **Create new project** 클릭

프로젝트 생성에 1~2분 정도 걸려요.

## 3단계: API 키 가져오기

서비스와 연결하려면 API 키가 필요해요.

1. 프로젝트 대시보드 → 왼쪽 사이드바 **Settings** (톱니바퀴 아이콘)
2. **API** 메뉴 클릭
3. 두 가지 값을 복사해두세요:
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon public key**: `eyJhbGci...`로 시작하는 긴 문자열

이 값들을 `.env.local`에 넣어요:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

## 4단계: 테이블 만들기

데이터를 저장할 테이블(엑셀의 시트처럼)을 만들어야 해요.

### SQL 편집기 사용하기

1. 왼쪽 사이드바 **SQL Editor** 클릭
2. 아래 같은 SQL을 입력하고 **Run** 클릭

예시: 뉴스 저장 테이블

```sql
CREATE TABLE news_items (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL,
  summary_ko TEXT,
  source_name TEXT,
  source_url TEXT,
  priority TEXT DEFAULT 'community',
  category TEXT[] DEFAULT ARRAY['general'],
  published_date DATE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

<Callout type="tip">
테이블 구조를 직접 짜기 어려우면 Claude Code에게 맡기세요.
```
> 뉴스 기사를 저장하는 Supabase 테이블 SQL을 만들어줘.
  저장할 항목: 제목, 요약, 출처, URL, 카테고리, 날짜
```
</Callout>

### Table Editor 사용하기 (더 쉬운 방법)

SQL이 낯설다면 GUI로 만들 수도 있어요:

1. 왼쪽 사이드바 **Table Editor** 클릭
2. **New Table** 클릭
3. 컬럼(열) 이름과 타입을 직접 입력

## 5단계: RLS(보안) 설정하기

Supabase는 기본적으로 **Row Level Security(RLS)**가 활성화되어 있어요. 이 때문에 초기에 데이터가 안 보이거나 저장이 안 되는 일이 생겨요.

> **비유**: RLS는 금고예요. 금고를 만들었는데 열쇠 정책(Policy)을 안 만들면 본인도 못 열어요. 정책을 만들어서 누가 데이터를 읽고 쓸 수 있는지 정해줘야 해요.

**가장 기본적인 정책 (서비스 초기 설정):**

```sql
-- 누구나 읽기 가능
CREATE POLICY "Allow public read"
ON 테이블이름 FOR SELECT
USING (true);

-- 누구나 쓰기 가능 (API로만 접근하는 경우)
CREATE POLICY "Allow insert"
ON 테이블이름 FOR INSERT
WITH CHECK (true);
```

```
> Supabase에서 news_items 테이블의 RLS 정책을 만들어줘.
  읽기는 누구나, 쓰기는 API 호출로만 가능하게.
```

## 6단계: 서비스 코드에서 연결하기

Claude Code에게 이렇게 말하면 연결 코드를 만들어줘요:

```
> Supabase 연결해서 news_items 테이블에서
  최신 10개 항목을 가져오는 코드 만들어줘
```

생성되는 코드:

```typescript
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)

const { data } = await supabase
  .from('news_items')
  .select('*')
  .order('created_at', { ascending: false })
  .limit(10)
```

직접 코드를 쓸 필요 없이 Claude Code가 알아서 작성해줘요.

## Supabase에서 데이터 확인하기

데이터가 잘 저장되고 있는지 확인하는 방법:

1. Supabase 대시보드 → **Table Editor**
2. 테이블 클릭
3. 저장된 행(데이터) 확인

또는 **SQL Editor**에서:

```sql
SELECT * FROM news_items ORDER BY created_at DESC LIMIT 10;
```

## 자주 발생하는 문제들

### 데이터가 저장되지 않을 때
→ RLS 정책 확인. INSERT 정책이 있는지 확인하세요.

### 데이터가 조회되지 않을 때
→ RLS 정책 확인. SELECT 정책이 있는지 확인하세요.

### "duplicate key" 에러
→ 같은 값을 두 번 넣으려 할 때. `upsert` + `ignoreDuplicates: true` 사용.

```
> Supabase에 데이터 저장할 때 중복이면 그냥 건너뛰도록 수정해줘
```

| 파일 저장 | 1GB |
| 동시 접속 | 200개 |

소규모 서비스는 무료 플랜으로 충분해요.
