# 뉴스레터 자동으로 만들고 발송하기

뉴스레터는 가장 ROI가 높은 마케팅 채널이에요. 하지만 매주 큐레이션하고 작성하는 게 부담스러웠다면, 이제 자동화해봐요. Claude Code가 최신 콘텐츠를 찾아서 뉴스레터 초안까지 만들어줍니다.

<Callout type="tip">
완전 자동화보다는 "90% 자동 + 10% 인간 편집" 방식을 추천해요. AI가 큐레이션하고 초안을 쓰면, 여러분이 브랜드 목소리를 입히고 발송하는 방식이에요.
</Callout>

---

## 1단계: 뉴스레터 설정 파일 만들기

```
> 뉴스레터 자동화 설정 파일을 만들어줘.
  
  newsletter-config.json:
  {
    "name": "[뉴스레터 이름]",
    "tagline": "[1줄 소개]",
    "frequency": "weekly",
    "send_day": "목요일",
    "topics": ["AI 트렌드", "마케팅 인사이트", "비즈니스 성장"],
    "sections": [
      "이번 주 핵심 뉴스 (3개)",
      "인사이트 한 줄",
      "추천 도구",
      "이번 주 팁"
    ],
    "tone": "친근하고 실용적"
  }
```

---

## 2단계: 콘텐츠 자동 큐레이션

```
> newsletter-config.json을 읽고
  Tavily로 이번 주 관련 콘텐츠를 수집해줘.
  
  각 토픽별 최신 뉴스/아티클 5개 검색.
  
  결과를 평가해서:
  - 가장 중요한 것 3개 선정
  - 왜 구독자에게 유용한지 1줄 설명
  - 원본 링크 포함
  
  curated-[날짜].md로 저장.
```

---

## 3단계: 뉴스레터 초안 작성

```
> curated-[날짜].md와 newsletter-config.json을 읽고
  이번 주 뉴스레터 초안을 작성해줘.
  
  형식: HTML 이메일
  
  구조:
  1. 헤더 (로고 + 날짜)
  2. 인트로 (2-3줄, 이번 주 핵심 주제)
  3. 이번 주 핵심 뉴스 (3개, 각 100자 요약 + 링크)
  4. 인사이트 한 줄 (주목할 만한 통계나 인용)
  5. 추천 도구 (1개, 간단한 소개)
  6. 이번 주 팁 (1개, 바로 쓸 수 있는 것)
  7. 푸터 (구독 취소 링크 포함)
  
  newsletter-[날짜].html로 저장.
```

---

## 4단계: 구독자 관리 시스템

```
> Supabase로 뉴스레터 구독자 관리 시스템을 만들어줘.
  
  테이블: subscribers
  컬럼: id, email, name, subscribed_at, status (active/unsubscribed), tags
  
  기능:
  - 구독 신청 API 엔드포인트
  - 구독 취소 처리 (원클릭)
  - 태그 기반 세그먼트 관리
  
  RLS 정책: 본인 데이터만 수정 가능
```

---

## 5단계: Resend로 발송 자동화

```
> Resend API를 사용해서 뉴스레터를 발송하는
  스크립트를 만들어줘.
  
  scripts/send-newsletter.ts
  
  기능:
  1. Supabase에서 active 구독자 목록 조회
  2. newsletter-[날짜].html 읽기
  3. 배치 발송 (시간당 100명, API 제한 준수)
  4. 발송 결과 로그 저장
  5. 발송 완료 후 Slack 알림
  
  환경변수: RESEND_API_KEY, SUPABASE_SERVICE_KEY
```

---

## 6단계: 구독 랜딩 페이지

```
> 뉴스레터 구독 랜딩 페이지를 만들어줘.
  
  포함 요소:
  - 뉴스레터 소개 (어떤 내용, 얼마나 자주)
  - 구독자 수 표시
  - 이전 호 미리보기 링크
  - 이메일 입력 폼
  - 구독 확인 이메일 자동 발송
  
  Vercel에 배포하고 URL 알려줘.
```

  </Before>

  </After>
</BeforeAfter>

## 성과 분석 자동화

```
> Resend 대시보드 데이터를 분석해줘.
  
  파일: resend-analytics.csv
  분석 항목:
  - 오픈율 트렌드
  - 클릭률 높은 섹션
  - 구독 취소율이 높았던 호와 공통점
  - A/B 테스트 결과 (제목 라인)
  - 다음 호 개선 포인트
```
