# Figma + Notion MCP 연동하기

MCP를 연결하면 Claude Code가 Figma 파일을 직접 열어 보거나, Notion 페이지를 읽고 쓸 수 있어요. 이 두 연동이 되면 디자이너와 PM의 워크플로우가 완전히 달라집니다.

## MCP가 뭔가요? (간단 복습)

MCP(Model Context Protocol)는 Claude Code가 외부 서비스에 접근할 수 있게 연결해주는 플러그인이에요.

> **비유**: 스마트폰의 앱처럼, MCP는 Claude Code의 능력을 확장하는 모듈이에요. Figma MCP를 설치하면 Claude Code가 Figma를 '손'처럼 다룰 수 있고, Notion MCP를 설치하면 Notion을 직접 읽고 쓸 수 있어요.

---

## Figma MCP 연동하기

### Figma MCP로 할 수 있는 것

- Figma 디자인을 읽어서 코드로 변환
- 컴포넌트 구조 분석
- 디자인 스펙 자동 추출
- 변수/컬러 토큰 확인

### 1. Figma API 토큰 발급받기

1. [figma.com](https://figma.com) 로그인
2. 오른쪽 위 프로필 아이콘 클릭 → **Settings**
3. **Security** 탭 클릭
4. **Personal access tokens** 섹션에서 **Generate new token**
5. 이름 입력 (예: `claude-code`) → **Generate token**
6. 생성된 토큰 복사 (이 창을 닫으면 다시 볼 수 없어요!)

### 2. Figma MCP 설치 및 설정

터미널에서:

```bash
claude mcp add figma
```

또는 Claude Code 안에서:

```
> Figma MCP를 설치해줘. API 토큰은 [복사한 토큰] 이야.
```

설정 확인:

```bash
claude mcp list
```

### 3. Figma MCP 사용하기

연결 후 Figma URL을 붙여넣으면 돼요:

```
> 이 Figma 파일의 메인 컴포넌트를 분석해줘:
  https://www.figma.com/design/xxxxx/내파일명
```

```
> 이 Figma URL의 버튼 컴포넌트를
  Next.js + Tailwind 코드로 만들어줘:
  https://www.figma.com/design/xxxxx?node-id=1-2
```

```
> 이 디자인의 컬러 팔레트와 폰트 스펙을 정리해줘
```

<Callout type="tip">
Figma에서 특정 컴포넌트의 링크를 복사하려면: 해당 요소 우클릭 → **Copy link to selection**
</Callout>

### 실제 활용 예시 (디자이너)

```
> 이 Figma 컴포넌트를 분석해서 React 컴포넌트로 만들어줘.
  우리 프로젝트의 기존 스타일 규칙도 유지해줘.
  [Figma URL]
```

```
> Figma에서 이 페이지의 전체 레이아웃을 파악하고
  Next.js 페이지 파일로 만들어줘
```

## Notion MCP 연동하기

### Notion MCP로 할 수 있는 것

- Notion 페이지 읽기/쓰기
- 데이터베이스에 새 항목 추가
- 회의록 자동 정리 후 Notion에 저장
- 프로젝트 현황을 Notion 페이지로 업데이트

### 1. Notion Integration 만들기

1. [notion.so/my-integrations](https://www.notion.so/my-integrations) 접속
2. **New integration** 클릭
3. 이름 입력 (예: `Claude Code`)
4. **Submit** 클릭
5. **Internal Integration Token** 복사 (`secret_...`로 시작)

### 2. Notion 페이지에 Integration 연결하기

**중요**: Integration을 만들었다고 모든 페이지에 접근되는 게 아니에요. 접근을 허용할 페이지마다 연결해야 해요.

1. 접근 허용할 Notion 페이지 열기
2. 오른쪽 위 **⋯** 메뉴 → **Connections**
3. 방금 만든 Integration 이름 검색 후 연결

<Callout type="info">
하위 페이지에는 자동으로 상속돼요. 최상위 워크스페이스 페이지에 연결하면 그 아래 모든 페이지에 접근 가능해요.
</Callout>

### 3. Notion MCP 설치 및 설정

```bash
claude mcp add -e NOTION_API_KEY=secret_xxx notion -- npx -y @notionhq/notion-mcp-server
```

또는 Claude Code 안에서:

```
> Notion MCP를 설치해줘. API 키는 secret_xxx야.
```

### 4. Notion MCP 사용하기

```
> Notion에서 "프로젝트 현황" 페이지를 읽어서
  이번 주 완료된 태스크를 요약해줘
```

```
> 오늘 회의 내용을 정리해서
  Notion "회의록" 데이터베이스에 새 페이지로 저장해줘.
  제목: [날짜] 주간 미팅
```

```
> Notion의 "콘텐츠 캘린더" 데이터베이스에
  다음 주 포스팅 일정 5개를 추가해줘
```

### 실제 활용 예시 (PM)

```
> Notion "로드맵" 페이지를 읽고
  이번 분기 완료된 기능과 남은 기능을 정리해줘
```

```
> 오늘 스크럼 회의록 파일(meeting.txt)을 읽고
  액션 아이템을 추출해서 Notion 태스크 DB에 추가해줘.
  담당자 필드도 채워줘.
```

## 두 MCP를 함께 쓰는 워크플로우

Figma와 Notion을 모두 연결하면 강력한 자동화가 가능해요:

**디자이너 워크플로우:**
```
> Figma에서 이 컴포넌트 디자인을 읽어서
  코드로 변환하고, 스펙 문서를 Notion에 저장해줘
  [Figma URL]
```

**PM 워크플로우:**
```
> Notion "기능 스펙" 페이지를 읽고
  개발팀에 전달할 기술 요구사항 문서를 만들어줘
```
