# 실전 활용 사례

실제로 비개발자들이 Claude Code를 어떤 식으로 활용하는지 업무 유형별로 살펴볼게요.

## 문서 및 콘텐츠 작업

### 대량 문서 처리

```
> contracts 폴더 안의 모든 계약서 파일에서 "갑"을 "발주처"로 바꿔줘
```

```
> 이 폴더의 모든 .docx 파일 목록과 각 파일의 수정 날짜를 보여줘
```

### 콘텐츠 정리 및 요약

```
> meeting_notes 폴더의 지난 달 회의록을 모두 읽고
  핵심 결정 사항만 모아서 monthly_summary.md로 만들어줘
```

```
> 이 긴 보고서를 3페이지 분량의 요약본으로 만들어줘
```

### 파일 형식 변환 준비

```
> 이 CSV 파일을 보기 좋게 정리하고, 헤더 행을 한국어로 바꿔줘
```

## 데이터 파일 작업

### CSV/데이터 분석

```
> sales.csv 파일을 읽고 월별 총 매출을 계산해줘
```

```
> customers.csv에서 서울 지역 고객만 필터링해서 seoul_customers.csv로 저장해줘
```

```
> 이 두 CSV 파일에서 중복되는 이메일 주소를 찾아줘
```

<Callout type="tip">
Claude Code는 CSV 파일을 직접 읽고 분석할 수 있어요. 엑셀 없이도 데이터를 처리할 수 있습니다.
</Callout>

### 파일 정리

```
> downloads 폴더에서 2023년보다 오래된 파일 목록을 보여줘
```

```
> 중복된 파일 이름이 있는지 찾아줘
```

## 웹사이트 수정 (HTML)

코딩을 전혀 몰라도 Claude Code로 웹사이트 파일을 수정할 수 있어요.

```
> index.html 파일에서 메뉴에 "서비스 소개" 항목을 추가해줘
```

```
> 웹사이트 모든 HTML 파일에서 푸터의 저작권 연도를 2024로 바꿔줘
```

```
> contact.html의 전화번호와 이메일 주소를 새로운 것으로 바꿔줘:
  전화: 02-1234-5678, 이메일: new@company.com
```

<BeforeAfter>
  <Before>
    웹 에이전시에 연락해서 수정 요청 → 며칠 후 변경 완료 → 비용 발생
  </Before>
  <After>
    Claude Code에게 직접 요청 → 5분 이내 변경 완료 → 무료
  </After>
</BeforeAfter>

## 업무 자동화

### 반복 작업

```
> photos 폴더 안의 모든 이미지 파일 이름 앞에 날짜 "20240326_"을 붙여줘
```

```
> 이 폴더에 있는 모든 .txt 파일을 하나의 파일로 합쳐줘.
  각 파일 이름을 구분선으로 넣어줘.
```

### 보고서 생성

```
> projects 폴더를 읽고 각 프로젝트의 상태를 정리한 주간 보고서를 만들어줘
```

## 개발자와 협업할 때

개발자가 만든 코드를 이해해야 할 때도 Claude Code가 도움이 돼요.

```
> 개발자가 보내준 이 코드 파일이 어떤 역할을 하는지 쉽게 설명해줘
```

```
> 이 설정 파일에서 서버 주소 부분이 어디인지 찾아줘
```

```
> 개발팀에서 준 이 에러 메시지가 무슨 뜻인지 설명해줘
```

## 학습과 탐구

```
> 이 폴더의 코드를 읽고 이 프로젝트가 어떻게 동작하는지 설명해줘
```

```
> package.json 파일을 보고 이 프로젝트에 어떤 기술이 사용됐는지 알려줘
```

## 웹사이트/서비스 제작

Claude Code로 단순한 파일 수정을 넘어 **처음부터 완전한 웹사이트나 서비스를 만들 수** 있어요.

이 claude-guide 사이트 자체가 좋은 예시입니다. Next.js 기반 앱, 13개 학습 페이지, 퀴즈 기능, Vercel 자동 배포까지 — **코딩을 한 줄도 직접 쓰지 않고** Claude Code에게 말로 시켜서 완성했어요.

```
> Next.js로 학습 가이드 사이트를 만들어줘.
  페이지마다 퀴즈 기능이 있고,
  사이드바에 목차가 나오는 구조로 만들어줘.
```

```
> 지금 만든 사이트를 Vercel에 배포할 수 있게 설정해줘.
  GitHub 저장소 연결도 같이 해줘.
```

```
> 홈페이지에 방문자가 진행 상황을 추적할 수 있는
  'N/13 완료' 형태의 진행 바를 추가해줘.
```

<Callout type="tip">
"코딩 없이 만든다"는 게 마법처럼 들리지만, Claude Code가 실제로 코드를 작성합니다. 여러분은 무엇을 만들지 말로 설명하고, Claude Code가 필요한 파일과 코드를 모두 생성해요. 이 사이트가 바로 그 증거입니다.
</Callout>

## 이력서/문서 PDF 자동 생성

이력서, 제안서, 보고서처럼 **깔끔하게 인쇄 가능한 문서**가 필요할 때 Claude Code를 활용하면 됩니다. HTML로 레이아웃을 잡고 PDF로 변환하는 방식이에요.

```
> 내 이력서 내용을 바탕으로 깔끔한 HTML 이력서를 만들어줘.
  인쇄하면 A4 1장에 딱 맞게 레이아웃을 잡아줘.
  이름: 홍길동, 경력: ..., 기술: ...
```

```
> resume.html 파일을 PDF로 변환해줘.
  여백은 위아래 20mm, 좌우 15mm로 설정해줘.
```

```
> 이 제안서 HTML 파일에 회사 로고 이미지를 헤더에 넣어줘.
  로고 파일은 logo.png야.
```

<Callout type="tip">
HTML → PDF 변환은 터미널에 puppeteer나 wkhtmltopdf 같은 도구가 설치돼 있어야 해요. 설치가 안 됐다면 "PDF 변환 도구를 설치하는 방법도 알려줘"라고 함께 요청하면 Claude Code가 안내해 줍니다.
</Callout>

## PRD·기획 문서 작성

신제품 기획서, 서비스 요구사항 문서(PRD), 프로젝트 제안서처럼 **구조화된 기획 문서**를 Claude Code가 직접 파일로 작성해 줍니다.

```
> 우리 팀이 새로 만들 구독 관리 앱의 PRD를 작성해줘.
  대상 사용자는 소규모 SaaS 팀이고,
  핵심 기능은 구독 현황 추적, 비용 분석, 갱신 알림이야.
  PRD.md 파일로 저장해줘.
```

```
> 방금 만든 PRD를 바탕으로 개발팀에 줄 기술 스펙 문서도 만들어줘.
  API 엔드포인트 목록과 데이터 구조도 포함해줘.
```

```
> 경쟁 서비스 3개(Notion, Airtable, Monday.com)의 기능을 비교한
  경쟁 분석 표를 만들어서 competitor-analysis.md로 저장해줘.
```

<Callout type="tip">
문서를 파일로 저장해 달라고 명시하는 게 중요해요. "만들어줘"만 하면 채팅창에 보여주고 끝나지만, "파일로 저장해줘"라고 하면 실제로 .md나 .docx 파일이 생성됩니다.
</Callout>

## 데이터베이스 연결 서비스

Supabase MCP를 설정하면 Claude Code가 데이터베이스에 직접 연결해서 테이블 생성, 데이터 조회, 수정까지 할 수 있어요. SQL을 몰라도 됩니다.

```
> Supabase에 고객 관리용 테이블을 만들어줘.
  컬럼은 이름, 이메일, 가입일, 구독 등급(free/pro/enterprise)이야.
```

```
> customers 테이블에서 지난 30일 안에 가입한 pro 등급 고객 목록을 조회해줘.
```

```
> 이 CSV 파일의 데이터를 Supabase products 테이블에 일괄 추가해줘.
```

<Callout type="tip">
Supabase MCP 연결은 처음 한 번만 설정하면 돼요. 설정 방법은 이 가이드의 'MCP 도구 연결' 페이지에서 확인하세요. 연결 후에는 "데이터베이스에서 ~해줘"라고 자연어로 요청하면 Claude Code가 알아서 SQL을 작성하고 실행합니다.
</Callout>

## 실제 업무에 바로 적용하는 팁

1. **작게 시작하세요**: 처음엔 한두 개 파일로 테스트해보세요
2. **구체적으로 요청하세요**: "수정해줘" 보다 "A를 B로 바꿔줘"가 더 정확해요
3. **확인하세요**: 변경 전에 Claude가 보여주는 내용을 꼭 검토하세요
4. **반복 활용하세요**: 같은 패턴의 작업은 요청 문구를 저장해두면 편해요
