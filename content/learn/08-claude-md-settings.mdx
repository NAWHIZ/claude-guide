# CLAUDE.md와 설정 커스터마이징

Claude Code를 더 똑똑하게 만드는 비결이 있어요. 바로 `CLAUDE.md` 파일이에요. 이 파일 하나로 Claude Code가 매번 프로젝트를 새로 파악하지 않고, 처음부터 맥락을 알고 일해줘요.

## CLAUDE.md가 뭔가요?

`CLAUDE.md`는 프로젝트 폴더 안에 넣어두는 특별한 텍스트 파일이에요. Claude Code는 대화를 시작할 때 이 파일을 자동으로 읽어서 프로젝트 맥락으로 사용해요.

비유하자면, 새로운 직원에게 주는 온보딩 문서 같은 거예요. "이 프로젝트는 이런 것이고, 이런 규칙을 따르고, 이런 방식으로 작업해줘"라는 내용을 미리 적어두면, Claude가 그걸 읽고 처음부터 맥락을 파악한 상태로 일해줘요.

### 없을 때 vs 있을 때

**CLAUDE.md가 없을 때**: Claude Code를 실행할 때마다 "이 프로젝트는 어떤 건가요?", "어떤 규칙을 따라야 하나요?"를 설명해야 해요.

**CLAUDE.md가 있을 때**: 설명 없이 바로 작업 요청을 시작할 수 있어요. Claude가 이미 프로젝트를 알고 있는 것처럼 행동해요.

## CLAUDE.md 파일 만들기

CLAUDE.md 파일은 일반 텍스트 파일이에요. 마크다운 형식(# 제목, - 목록 등)으로 작성하면 더 잘 읽혀요.

프로젝트 폴더의 최상위 위치에 `CLAUDE.md`라는 이름으로 저장하면 돼요.

Claude Code에게 만들어달라고 해도 되고, 직접 텍스트 편집기로 만들어도 돼요.

```
> CLAUDE.md 파일을 만들어줘. 이 프로젝트에 대한 기본 정보를 담아줘
```

## CLAUDE.md 작성 예시

### 개인 블로그 프로젝트

```markdown
# 내 블로그 프로젝트

## 프로젝트 소개
Jekyll로 만든 개인 기술 블로그입니다.
주로 마케팅과 일상 이야기를 다룹니다.

## 폴더 구조
- _posts/: 블로그 포스트 (마크다운 파일)
- _drafts/: 작성 중인 초안
- images/: 이미지 파일
- _config.yml: 사이트 설정

## 작업 규칙
- 파일 수정 전에 반드시 백업본을 먼저 만들어주세요
- 모든 답변은 한국어로 해주세요
- 코드나 기술적인 내용은 비개발자가 이해하기 쉽게 설명해주세요
- 포스트 파일명은 YYYY-MM-DD-제목.md 형식으로 해주세요

## 자주 하는 작업
- 새 포스트 초안 작성
- 기존 포스트 내용 편집
- 이미지 파일명 정리
```

### 업무 문서 관리 프로젝트

```markdown
# 마케팅팀 문서 폴더

## 용도
마케팅팀의 각종 문서, 보고서, 기획안을 관리하는 폴더입니다.

## 폴더 구조
- reports/: 주간/월간 보고서
- campaigns/: 캠페인 기획안
- templates/: 자주 쓰는 문서 템플릿

## 규칙
- 파일명에는 날짜를 포함해주세요 (예: 2026-03-26_주간보고서.md)
- 회사명, 제품명, 담당자 이름 등 민감한 정보는 수정하지 말고 확인을 요청해주세요
- 외부 공유용 문서는 내부 용어를 쉬운 말로 바꿔주세요

## 자주 쓰는 작업
- 보고서 초안 작성
- 문서 내용 요약
- 템플릿 기반으로 새 문서 만들기
```

<Callout type="info">
CLAUDE.md는 팀원들과 공유할 때도 매우 유용해요. 팀 전체가 같은 CLAUDE.md를 사용하면 모든 팀원이 동일한 맥락과 규칙으로 Claude Code를 사용할 수 있어요. Git 같은 버전 관리 시스템에 포함시켜 공유하면 좋아요.
</Callout>

## CLAUDE.md에 포함하면 좋은 내용들

### 필수 항목
- **프로젝트 소개**: 이 폴더/프로젝트가 무엇인지 한두 문장으로
- **중요한 파일/폴더**: 어떤 파일들이 있고 각각 어떤 역할인지
- **작업 규칙**: 파일을 수정할 때 따라야 할 규칙

### 선택 항목
- 자주 하는 작업 목록
- 사용하는 용어 정의 (회사 내부 용어 등)
- 피해야 할 것들 (수정하면 안 되는 파일 등)
- 선호하는 답변 스타일 (한국어, 간결하게, 상세하게 등)

## 개인 프로젝트용 CLAUDE.md 템플릿

아래 템플릿을 복사해서 내 프로젝트에 맞게 수정해보세요.

```markdown
# [프로젝트 이름]

## 프로젝트 소개
[이 프로젝트가 무엇인지 2-3문장으로 설명]

## 폴더 구조
- [폴더명/파일명]: [설명]
- [폴더명/파일명]: [설명]

## 작업 규칙
- 파일 수정 전 백업 생성
- 한국어로 답변
- [기타 규칙]

## 하면 안 되는 것
- [수정하면 안 되는 파일 또는 작업]

## 자주 하는 작업
- [작업 1]
- [작업 2]
```

## settings.json으로 추가 설정하기

CLAUDE.md 외에도 `.claude/settings.json` 파일로 Claude Code의 동작을 더 세밀하게 설정할 수 있어요. 이 파일은 선택 사항이에요.

```json
{
  "theme": "dark",
  "autoApprove": ["read", "list"]
}
```

- `theme`: 화면 테마 (dark / light)
- `autoApprove`: 자동으로 허용할 작업 유형 (매번 확인 없이 진행)

이 파일은 Claude Code에게 만들어달라고 요청해도 되고, 직접 작성해도 돼요.
