# 막혔을 때 대처법

Claude Code를 쓰다 보면 반드시 막히는 순간이 와요. 에러 메시지, 배포 실패, 데이터가 안 보이는 상황... 이 페이지를 북마크해두고 막힐 때마다 찾아오세요.

<Callout type="tip">
**황금 법칙**: 에러가 났을 때 "왜 안 돼?"보다 **에러 메시지 전체를 그대로 복사해서 Claude Code에게 붙여넣기**하는 게 훨씬 빠르게 해결돼요.
</Callout>

---

## Claude Code 자체 문제

### Claude Code가 엉뚱한 걸 수정했어요

```
> 방금 수정한 내용을 되돌려줘
```

또는 `/undo`로 마지막 변경을 취소하세요. 중요한 파일이라면 항상 작업 전에 백업을 요청하는 게 안전해요.

```
> 이 파일 수정하기 전에 backup/ 폴더에 복사해줘
```

### 응답이 느리거나 중간에 잘려요

대화가 너무 길어진 거예요. `/compact`로 압축하세요.

```
> /compact
```

그래도 안 되면 `/clear`로 새로 시작하세요. 이전 내용은 CLAUDE.md에 정리해두면 다음 대화에서도 이어갈 수 있어요.

### "요청 한도를 초과했어요" 메시지

플랜 한도에 도달한 거예요. 몇 시간~하루 기다리면 리셋돼요. 매일 반복되면 Max 플랜 업그레이드를 고려하세요.

### Claude Code가 계속 같은 실수를 반복해요

CLAUDE.md에 규칙을 추가하세요.

```
> CLAUDE.md에 이 규칙 추가해줘:
  앞으로 파일 수정할 때는 항상 수정 내용을 먼저 설명하고 동의를 구한 후 진행해
```

## GitHub 문제

### "git push" 실패 — 인증 오류

```
remote: Invalid username or password
```

GitHub Personal Access Token이 만료됐거나 없는 거예요.

**해결**:
1. GitHub → Settings → Developer Settings → Personal access tokens → 새 토큰 발급
2. 터미널에서 다시 push 시도 → 비밀번호 입력 시 토큰 붙여넣기

또는 Claude Code에게:
```
> GitHub push가 인증 오류로 실패해. 해결 방법 알려줘
```

### "git push" 실패 — 충돌(conflict)

```
error: failed to push some refs
```

다른 곳에서 같은 파일을 수정했을 때 발생해요.

```
> git push 충돌이 났어. 해결해줘
  에러: [에러 메시지 붙여넣기]
```

## Vercel 배포 문제

### 빌드 에러로 배포 실패

Vercel 대시보드 → Deployments → 해당 배포 클릭 → **Build Logs** 탭에서 에러 확인

```
> Vercel 빌드 로그에서 이런 에러가 났어. 고쳐줘:
  [에러 메시지 전체 붙여넣기]
```

### 배포는 됐는데 사이트가 작동 안 해요

환경변수가 빠진 경우가 많아요.

**확인 순서**:
1. Vercel 대시보드 → 프로젝트 → **Settings → Environment Variables**
2. `.env.local`에 있는 변수들이 Vercel에도 다 있는지 확인
3. 없으면 추가 후 **Redeploy**

```
> Vercel에 환경변수 등록을 빠뜨렸어.
  .env.local 파일을 읽고 어떤 변수를 추가해야 하는지 알려줘
```

### 배포 후 API가 작동 안 해요 (500 에러)

Vercel 대시보드 → **Logs** 탭에서 런타임 에러 확인

```
> Vercel Logs에서 이런 에러가 나왔어:
  [에러 메시지 붙여넣기]
  어떻게 고치면 돼?
```

## Supabase 문제

### 데이터가 저장이 안 돼요

RLS(보안 정책) 문제가 가장 흔해요.

**확인 방법**: Supabase → SQL Editor에서 실행:
```sql
SELECT * FROM pg_policies WHERE tablename = '테이블이름';
```

정책이 없으면:
```
> Supabase에 데이터 저장이 안 돼. RLS 정책 문제인 것 같아.
  [테이블이름] 테이블에 누구나 읽고 쓸 수 있는 정책 만들어줘
```

### 데이터가 조회가 안 돼요 (빈 배열 반환)

1. Supabase 대시보드 → **Table Editor**에서 실제 데이터가 있는지 확인
2. RLS SELECT 정책이 있는지 확인
3. 환경변수(URL, API 키)가 맞는지 확인

```
> Supabase에서 데이터를 가져오는데 빈 배열이 와.
  연결 설정과 RLS 정책을 확인하고 고쳐줘
```

### "duplicate key" 에러

같은 데이터를 두 번 넣으려 할 때예요.

```
> Supabase에 중복 데이터가 저장될 때 에러 대신 그냥 건너뛰도록 수정해줘
```

### Supabase 연결 자체가 안 돼요

```
> Supabase 연결이 안 돼. 환경변수 설정을 확인해줘
```

`.env.local` 파일에서:
- `NEXT_PUBLIC_SUPABASE_URL` — `https://`로 시작해야 해요
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — `eyJ`로 시작하는 긴 문자열이어야 해요

## MCP 연결 문제

### MCP 서버가 연결 안 돼요

```
> claude mcp list
```
로 현재 상태 확인. 연결 실패한 서버가 있으면:

```
> [서버이름] MCP 연결이 안 돼. 재설치 도와줘
```

### Notion MCP로 페이지를 못 읽어요

Notion은 페이지마다 Integration 연결이 필요해요.

**해결**: Notion 페이지 우측 상단 `...` → **Connections** → Claude Code Integration 연결

### Figma MCP가 파일을 못 읽어요

Personal Access Token이 만료됐을 수 있어요.

**해결**: Figma → Settings → Security → Personal access tokens → 새 토큰 발급 후 재설정

```
> Figma MCP 토큰을 새로 발급받았어. [새 토큰] 으로 재설정해줘
```

## 자주 보이는 에러 메시지 해석

| 에러 메시지 | 의미 | 해결 방향 |
|------------|------|-----------|
| `Cannot find module` | 필요한 패키지가 없음 | `npm install` 실행 |
| `TypeError: Cannot read properties of undefined` | 없는 데이터를 읽으려 함 | 데이터 존재 여부 확인 |
| `401 Unauthorized` | API 키가 없거나 잘못됨 | 환경변수 확인 |
| `403 Forbidden` | 접근 권한 없음 | RLS 정책 또는 API 키 권한 확인 |
| `404 Not Found` | URL이나 파일 경로가 틀림 | 경로 확인 |
| `500 Internal Server Error` | 서버에서 에러 발생 | 서버 로그 확인 |
| `CORS error` | 도메인 접근 허용 문제 | Supabase/서버 CORS 설정 확인 |
| `duplicate key value` | 중복 데이터 저장 시도 | upsert 또는 ignoreDuplicates 사용 |

---

## 그래도 해결이 안 될 때

1. **에러 메시지 전체를 Claude Code에 붙여넣기** — 가장 효과적
2. **관련 파일 이름을 같이 알려주기** — "app/page.tsx에서 에러가 나"
3. **언제부터 안 됐는지 알려주기** — "어제까지는 됐는데 오늘부터 안 돼"
4. **무엇을 바꿨는지 알려주기** — "환경변수 추가 후부터 안 돼"

```
> [파일명]에서 이런 에러가 났어:
  [에러 메시지 전체]
  
  어제까지는 잘 됐고, 오늘 [변경 내용]을 한 후부터 안 됐어.
  원인을 찾아서 고쳐줘.
```

<Callout type="tip">
Claude Code가 세 번 이상 같은 방법으로 시도해도 해결이 안 된다면 접근 방식 자체를 바꿔야 해요. "이 방법 말고 다른 접근 방법으로 시도해줘"라고 말해보세요.
</Callout>
