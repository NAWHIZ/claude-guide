# OMC 설치 및 설정

Claude Code를 기본으로 쓰는 것과 OMC(oh-my-claudecode)를 설치해서 쓰는 건 자전거와 오토바이의 차이예요. 이 가이드에서 설명하는 세팅이 완료되면 이 사이트를 만든 것과 동일한 환경이 갖춰집니다.

## OMC가 뭔가요?

OMC(oh-my-claudecode)는 Claude Code 위에 설치하는 **다중 에이전트 오케스트레이션 레이어**예요.

쉽게 말하면: Claude Code 하나만 쓰는 대신, 여러 전문가 AI를 동시에 지휘하는 시스템이에요.

> **비유**: 기본 Claude Code가 만능 직원 1명이라면, OMC는 전문 분야별 팀이에요. 기획자, 개발자, 디자이너, QA가 각자 전문 역할을 맡아서 동시에 일해요. 복잡한 프로젝트를 훨씬 빠르고 정확하게 처리합니다.

OMC가 해주는 것:
- **멀티 에이전트**: 기획, 구현, 검토를 각각 다른 AI가 담당
- **메모리 시스템**: 이전 대화 내용과 선호를 기억
- **스킬(Skills)**: `/autopilot`, `/ralph` 같은 고급 워크플로우
- **훅(Hooks)**: 도구 실행 전후 자동 검증
- **프로젝트 메모리**: CLAUDE.md 기반 프로젝트별 설정

## 설치하기

### 전제 조건

- Claude Code 설치 완료 (02번 아티클 참고)
- Node.js 18 이상
- 터미널 기본 사용법 숙지

### 설치 방법

```bash
npm install -g oh-my-claudecode
```

또는 Claude Code 안에서:

```
> oh-my-claudecode를 설치해줘
```

설치 확인:

```bash
omc --version
```

### 초기 설정

```bash
omc setup
```

또는 Claude Code에서:

```
> /oh-my-claudecode:omc-setup
```

설정 과정에서:
1. 기본 AI 모델 선택 (sonnet 권장)
2. 메모리 저장 위치 설정
3. 기본 스킬 활성화

## 핵심 스킬 소개

### /autopilot — 자동 기획부터 구현까지

복잡한 작업을 던지면 OMC가 알아서 기획 → 구현 → 검증 과정을 자동으로 진행해요.

```
> /autopilot 뉴스 자동 수집 기능을 추가해줘.
  Tavily API로 매일 아침 10개 기사를 수집하고
  Supabase에 저장하는 기능이야.
```

OMC가 하는 일:
1. 요구사항 분석 (analyst 에이전트)
2. 구현 계획 수립 (planner 에이전트)
3. 코드 작성 (executor 에이전트)
4. 코드 검토 (reviewer 에이전트)
5. 결과 검증 (verifier 에이전트)

### /ralph — 반복 실행 모드

완료될 때까지 멈추지 않고 계속 시도하는 모드예요.

```
> ralph 이 서비스의 빌드 에러를 고쳐줘.
  완전히 해결될 때까지 계속 시도해.
```

### /plan — 전략 수립

큰 프로젝트를 시작하기 전에 계획을 세우는 모드예요.

```
> /plan 개인 일정 관리 서비스를 만들고 싶어.
  사용자가 할 일을 입력하면 Notion에 자동 저장되고,
  매일 아침 Slack으로 오늘 할 일을 알려주는 서비스야.
```

OMC가 질문을 통해 요구사항을 구체화하고 단계별 계획을 작성해줘요.

## 메모리 시스템

OMC는 세 가지 메모리를 가져요:

### 1. 프로젝트 메모리 (`.omc/project-memory.json`)

현재 프로젝트에 대한 정보를 저장해요.

```
> 이 프로젝트는 마케팅 팀용 대시보드야.
  Supabase에 campaign_data 테이블이 있고,
  관리자만 데이터를 수정할 수 있어.
  이 정보를 프로젝트 메모리에 저장해줘.
```

이후 같은 프로젝트에서 작업할 때 이 맥락을 자동으로 참고해요.

### 2. 세션 메모리 (노트패드)

현재 작업 세션 동안 참고할 정보를 저장해요.

```
> 오늘 작업 목표:
  1. 로그인 페이지 완성
  2. 대시보드 차트 추가
  3. 배포
  이것을 노트패드에 저장해줘.
```

### 3. 글로벌 메모리 (`~/.claude/projects/memory/`)

모든 프로젝트에서 공통으로 쓰이는 개인 설정이에요.

```
> 앞으로 모든 프로젝트에서 커밋 메시지는 한국어로 써줘.
  이 설정을 기억해줘.
```

## CLAUDE.md 고급 설정

OMC와 함께 쓰면 CLAUDE.md가 더 강력해져요.

```
> 이 프로젝트의 CLAUDE.md를 업데이트해줘.
  아래 내용을 추가해:
  - 스택: Next.js 15, Supabase, Vercel
  - 코드 스타일: 한국어 주석, TypeScript strict
  - 배포 방식: GitHub push → Vercel 자동 배포
  - 금지 사항: console.log 배포 코드에 포함 금지
```

## 에이전트 라우팅 이해하기

OMC는 작업 유형에 따라 자동으로 적합한 에이전트를 선택해요:

| 작업 유형 | 담당 에이전트 |
|-----------|--------------|
| 버그 수정 | debugger |
| 새 기능 개발 | executor |
| 코드 리뷰 | code-reviewer |
| 아키텍처 설계 | architect |
| 보안 검토 | security-reviewer |
| 문서 작성 | writer |
| 테스트 작성 | test-engineer |

직접 지정도 가능해요:

```
> /oh-my-claudecode:security-reviewer
  이 API 엔드포인트에 보안 취약점이 있는지 검토해줘
```

## MCP와 OMC 함께 쓰기

OMC + MCP 조합이 완성되면 정말 강력해요:

```
> /autopilot
  Figma에서 이 컴포넌트를 읽어서 [URL]
  코드로 변환하고, 기존 프로젝트에 추가하고,
  GitHub에 올리고, Notion 작업 목록에 완료 표시해줘
```

이 한 문장으로:
1. Figma MCP → 디자인 읽기
2. executor → 코드 구현
3. git → GitHub 업로드
4. Notion MCP → 작업 완료 처리
5. Vercel → 자동 배포

까지 진행돼요.

## OMC 상태 확인하기

```bash
claude mcp list          # 연결된 MCP 목록
```

Claude Code 안에서:

```
> 현재 OMC 설정 상태를 알려줘
> 설치된 MCP와 활성화된 스킬 목록을 보여줘
```

## 문제 해결

### OMC 명령어가 인식되지 않을 때

```bash
npm install -g oh-my-claudecode
claude mcp list
```

### 에이전트가 응답하지 않을 때

```
> /oh-my-claudecode:cancel
```

현재 실행 중인 작업을 취소하고 초기화해요.
