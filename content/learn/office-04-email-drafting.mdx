# 상황별 이메일, AI가 초안 써줘

어려운 이메일 앞에서 멍하니 빈 화면을 바라본 적 있으세요? 사과해야 할 때, 거절해야 할 때, 조심스러운 제안을 해야 할 때... Claude Code에게 상황을 설명하면 바로 초안이 나와요.

<Callout type="tip">
AI가 쓴 초안을 그대로 보내지 마세요. 초안을 받고 본인의 목소리로 1-2문장 다듬는 것이 가장 좋아요. 70% 완성도 초안을 30초에 만들고, 나머지 30%는 직접 채우는 방식을 추천해요.
</Callout>

---

## 1단계: 이메일 상황 파일 만들기

자주 쓰는 이메일 유형과 내 스타일을 파일에 저장해두면:

```
> 내 이메일 스타일 가이드를 만들어줘.
  
  email-style.md에 저장:
  
  이름: [이름]
  직급: [직급]
  회사: [회사명]
  
  일반 원칙:
  - 톤: 정중하면서도 직접적
  - 길이: 짧게 (5문장 이내 선호)
  - 인사말 스타일: "안녕하세요, [이름]님"
  - 마무리: "감사합니다. [이름] 드림"
  - 피해야 할 표현: [표현 목록]
```

---

## 2단계: 상황별 이메일 초안 생성

**사과 이메일:**
```
> email-style.md를 참고해서 사과 이메일 초안을 써줘.
  
  상황: 납품 기한을 3일 지연했어. 
  원인: 부품 수급 문제.
  해결 계획: 다음 주 화요일 납품 예정.
  
  상대방: 주요 클라이언트 담당자
  기대 결과: 관계 유지하면서 이해 구하기
```

**협력 제안 이메일:**
```
> email-style.md를 참고해서 협력 제안 이메일 초안을 써줘.
  
  상황: 타 부서에 프로젝트 협력 요청
  배경: 우리 부서의 [프로젝트명] 진행 중
  요청 내용: 2주간 데이터 분석 지원
  제공할 것: 결과물 공동 발표 기회
  
  상대방: 데이터팀 팀장
```

**클레임 대응 이메일:**
```
> email-style.md를 참고해서 고객 클레임 대응 이메일을 써줘.
  
  고객 클레임: [클레임 내용 붙여넣기]
  
  우리 입장: 일부는 인정, 일부는 설명 필요
  제안할 해결책: [해결책]
  
  톤: 공감하면서도 명확하게
```

---

## 3단계: 이메일 개선 요청

초안이 마음에 들지 않으면:

```
> 이 이메일 초안을 개선해줘:
  [초안 붙여넣기]
  
  개선 방향:
  - 더 간결하게 (현재 너무 길다)
  - 핵심 요청을 앞에 배치
  - 덜 딱딱하게
```

---

## 4단계: 이메일 템플릿 라이브러리 만들기

자주 쓰는 이메일 유형을 템플릿으로 저장:

```
> 다음 유형의 이메일 템플릿을 만들어줘.
  각 템플릿은 [대괄호] 변수 포함.
  
  templates/emails/ 폴더에 저장:
  
  1. 미팅 요청.md
  2. 제안서 발송.md
  3. 일정 변경 요청.md
  4. 프로젝트 완료 보고.md
  5. 지연 안내.md
  6. 소개/추천 요청.md
```

---

## 5단계: 받은 이메일 분석 및 답장

받은 이메일을 붙여넣으면 대응 방법도 알려줘요:

```
> 이 이메일에 어떻게 답장할지 조언해줘:
  
  [받은 이메일 내용]
  
  나의 상황:
  - 요청을 수락하고 싶지만 조건이 있음
  - 협상이 필요한 부분: [내용]
  
  답장 초안도 함께 써줘.
```

---

## 실전: 영어 이메일도 바로

```
> email-style.md 스타일로 영어 비즈니스 이메일을 써줘.
  
  상황: 해외 파트너사에 납품 일정 변경 안내
  
  상대방: [이름], [회사명]
  
  한국어로도 같이 써줘 (내부 공유용).
```

<Callout type="info">
영어 이메일이 어려웠던 이유는 작성 시간이 아니라 자신감 부족이에요. Claude Code가 초안을 만들어주면 리뷰하고 수정하는 것만 하면 돼서 훨씬 쉬워요.
</Callout>
