## 기본 명령어

| 명령어 | 설명 |
|--------|------|
| `/help` | 사용 가능한 모든 명령어 목록 표시 |
| `/exit` | Claude Code 종료 |
| `/clear` | 현재 대화 기록 초기화 |
| `/compact` | 대화 내용 요약 압축 (컨텍스트 공간 확보) |
| `/status` | 현재 설정 및 API 키 상태 확인 |

## 모델 관련

| 명령어 | 설명 |
|--------|------|
| `/model` | 현재 사용 중인 AI 모델 확인 및 변경 |

사용 가능한 모델:

| 모델 | 특징 | 추천 용도 |
|------|------|-----------|
| `claude-opus-4-5` | 가장 강력 | 복잡한 분석, 긴 문서 작업 |
| `claude-sonnet-4-5` | 속도/성능 균형 | 일반 업무 (기본값 권장) |
| `claude-haiku-4-5` | 빠르고 저렴 | 간단한 질문, 빠른 편집 |

## 설정 관련

| 명령어 | 설명 |
|--------|------|
| `/config` | Claude Code 설정 보기 및 변경 |

## 권한 관련

Claude Code가 파일을 수정하거나 명령어를 실행하기 전에 확인을 요청해요.

| 응답 | 의미 |
|------|------|
| `y` 또는 엔터 | 허용 |
| `n` | 거부 |
| `a` | 이 세션에서 항상 허용 |

## 키보드 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl + C` | 현재 작업 중단 / 종료 |
| `Ctrl + L` | 화면 지우기 |
| `↑` / `↓` | 이전/다음 입력 기록 |
| `Shift + Enter` | 줄바꿈 (여러 줄 입력) |
| `Tab` | 파일/폴더 이름 자동완성 |

## 자주 묻는 슬래시 명령어 FAQ

**Q. `/clear`와 `/compact`의 차이는?**

`/clear`는 대화를 완전히 초기화해요. `/compact`는 대화 내용을 요약해서 공간을 절약하되 맥락은 유지해요.

**Q. 모델을 바꾸면 요금도 달라지나요?**

네. Opus가 가장 비싸고 Haiku가 가장 저렴해요. 일반 업무에는 Sonnet이 적당해요.

**Q. 실수로 파일을 수정했을 때 되돌리려면?**

Claude Code에 "방금 변경한 파일을 원래대로 되돌려줘"라고 요청하거나, git을 사용한다면 `git checkout`으로 되돌릴 수 있어요.
