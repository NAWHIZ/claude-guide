# 프롬프트 패턴 모음

Claude에게 더 좋은 결과를 얻기 위해 검증된 프롬프트 패턴들을 상황별로 정리했어요. 대괄호 `[...]` 안의 내용을 내 상황에 맞게 바꿔서 바로 사용해보세요.

---

## 역할 부여 패턴

역할을 주면 Claude가 그 전문가의 관점과 언어로 답변해요.

**전문가 역할**
```
당신은 [직업/전문 분야] 전문가입니다. [작업]을 해주세요.
```
예시: "당신은 10년 경력의 UX 디자이너입니다. 이 앱의 온보딩 흐름에서 개선할 점을 알려주세요."

**교사/설명자 역할**
```
[대상]도 이해할 수 있게 [개념]을 설명해주세요.
```
예시: "중학생도 이해할 수 있게 블록체인을 설명해주세요."

**편집자 역할**
```
다음 글을 꼼꼼한 편집자의 시각으로 검토하고, 어색한 표현과 논리적 허점을 찾아서 수정안을 제시해주세요.
```

**비판적 검토자 역할**
```
당신은 냉정한 투자자입니다. 아래 사업 아이디어의 약점과 위험 요소를 솔직하게 평가해주세요.
```

**악마의 변호인 역할**
```
아래 주장에 반대하는 입장에서 가장 강력한 반박 논거를 3가지 제시해주세요.
```

---

## 형식 지정 패턴

원하는 출력 형식을 명확히 지정하면 바로 활용 가능한 결과물이 나와요.

**표 형식 비교**
```
[항목 A], [항목 B], [항목 C]를 [기준1], [기준2], [기준3] 기준으로 비교하는 표를 만들어주세요.
```

**단계별 설명**
```
[작업/과정]을 단계별로 설명해주세요. 각 단계에 번호를 붙이고, 주의사항이 있으면 함께 표시해주세요.
```

**핵심 요약**
```
아래 내용을 [숫자]가지 핵심 포인트로 요약해주세요. 각 포인트는 한 문장으로.
```

**짧게 압축**
```
아래 내용을 트위터 한 줄(140자 이내)로 압축해주세요.
```

**Q&A 형식**
```
[주제]에 대해 초보자가 자주 묻는 질문 [숫자]개와 각 답변을 Q&A 형식으로 만들어주세요.
```

**1분 스피치**
```
아래 내용을 1분 스피치(약 200자) 형식으로 써주세요. 청중은 [대상]이에요.
```

## 맥락 제공 패턴

배경을 잘 전달할수록 맞춤형 답변이 나와요.

**대상 독자 지정**
```
다음 글을 [대상: 예) IT를 모르는 50대 임원]이 이해할 수 있는 수준으로 다시 써주세요.
```

**목적 명시**
```
이 글의 목적은 [목적: 예) 투자자를 설득하는 것]입니다. 이 목적에 맞게 아래 초안을 다듬어주세요.
```

**배경 + 질문**
```
[상황 설명: 예) 우리 팀은 5인 스타트업이고, 다음 달 첫 번째 투자 피칭이 있어요]. 이 상황에서 [질문]을 어떻게 해결하면 좋을까요?
```

**제약 조건 명시**
```
[작업]을 해주세요. 조건: (1) [조건1], (2) [조건2], (3) [조건3]. 반드시 포함할 것: [필수 요소]. 피해야 할 것: [금지 사항].
```

## 검토/개선 패턴

Claude가 자신의 답변 또는 내가 가져온 글을 개선하도록 시킬 수 있어요.

**논리 검토**
```
위 내용의 논리적 허점이나 근거가 부족한 부분을 찾아주세요. 개선 방법도 함께 제안해주세요.
```

**다른 관점**
```
위 내용을 [반대 입장 / 다른 이해관계자 / 비판적 시각]에서 바라보면 어떤 문제가 있을까요?
```

**개선 제안**
```
이 글을 더 [설득력 있게 / 간결하게 / 전문적으로 / 친근하게] 만들 방법 3가지를 구체적으로 제안해주세요.
```

**사실 확인 요청**
```
아래 내용에서 사실 여부를 확인해야 할 부분이 있으면 표시해주세요. 확실하지 않은 내용은 모른다고 해주세요.
```

**다시 써주기**
```
위 답변을 더 [짧게 / 구체적으로 / 쉬운 말로 / 강하게] 다시 써주세요.
```

## 창작 패턴

아이디어 발굴과 창작 작업에 유용한 패턴들이에요.

**브레인스토밍**
```
[주제]에 대한 아이디어를 [숫자]개 브레인스토밍해주세요. 실현 가능성보다 다양성을 우선으로, 엉뚱한 아이디어도 환영해요.
```

**다양한 버전 생성**
```
아래 내용을 [스타일1: 예) 유머러스하게], [스타일2: 예) 감성적으로], [스타일3: 예) 딱딱하게] 세 가지 버전으로 각각 써주세요.
```

**예시 기반 스타일 복제**
```
아래 예시와 동일한 스타일로 [새로운 주제]에 대해 써주세요.

[예시 텍스트]
```

**제목/슬로건 생성**
```
[브랜드/콘텐츠]의 [제목 / 슬로건 / 태그라인]을 [숫자]개 만들어주세요. 타겟은 [대상]이고, 핵심 메시지는 [메시지]예요.
```

## 빠른 활용 치트시트

| 상황 | 프롬프트 시작 문구 |
|------|-------------------|
| 이메일 작성 | "다음 상황에 맞는 [공손한/간결한/설득력 있는] 이메일을 써줘:" |
| 문서 요약 | "아래 내용을 [대상]을 위해 [형식]으로 요약해줘:" |
| 아이디어 발굴 | "[주제]에 대한 아이디어 [숫자]개를 다양한 각도에서 제안해줘" |
| 글 개선 | "아래 글을 더 [목표]하게 다듬어줘. 수정한 부분은 표시해줘:" |
| 개념 설명 | "[개념]을 [대상]이 이해할 수 있게 [비유/예시를 들어] 설명해줘" |
| 비교 분석 | "[A]와 [B]를 [기준]으로 비교해서 표로 정리해줘" |
| 검토 요청 | "아래 내용의 문제점과 개선 방법을 솔직하게 알려줘:" |

<Callout type="tip">패턴을 외울 필요는 없어요. 결과가 마음에 들지 않을 때마다 이 페이지로 돌아와서 더 구체적인 패턴을 적용해보세요. 쓰다 보면 자연스럽게 익혀지거든요.</Callout>
